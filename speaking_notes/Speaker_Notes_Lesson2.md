### 📄 หน้าที่ 31: เครื่องหมายคณิตศาสตร์ (Operators: + - * / %)

**🎤 บทพูด (สิ่งที่ต้องพูด):**
"เครื่องหมายคณิตศาสตร์ในภาษา C เราเรียกว่า 'Operators' ครับ ตัวหลักๆ ที่เราใช้บ่อยมากมีอยู่ 5 ตัวด้วยกัน"
"สี่ตัวแรก พี่เชื่อว่าทุกคนคุ้นเคยกันดีอยู่แล้ว คือ บวก (`+`), ลบ (`-`), คูณ (`*` ดอกจัน), และ หาร (`/` สแลช) ... สังเกตนะครับว่าเราไม่ใช้เครื่องหมายกากบาท `x` หรือเครื่องหมายหารแบบในหนังสือเรียน เพราะในคีย์บอร์ดมันไม่มี เราเลยใช้ `*` และ `/` แทน"
"แต่ตัวที่ 5 นี่สิครับ ไฮไลท์เลย! เครื่องหมาย `%` (เปอร์เซ็นต์) ในภาษา C มันไม่ได้แปลว่าร้อยละนะครับ! เราเรียกมันว่า **'Modulo' (มอดดิวโล) หรือเรียกสั้นๆ ว่า 'มอด'**"
"แล้วมันคืออะไร? มันคือ **การหารเอาเศษ** ครับ!"
"ลองนึกภาพตามพี่นะ สมมติพี่มีลูกอม 10 เม็ด จะแบ่งให้เด็ก 3 คน คนละเท่าๆ กัน ... ได้คนละกี่เม็ดครับ? (รอเด็กตอบ: 3 เม็ด) ใช่ครับ! แล้วเหลือเศษกี่เม็ด? (รอเด็กตอบ: 1 เม็ด)"
"ไอ้เจ้าเศษ 1 เม็ดนี่แหละครับ คือผลลัพธ์ของการทำ Modulo! ถ้าเราเขียนในโค้ดก็คือ `10 % 3 = 1`"
"เดี๋ยวเราจะได้เห็นกันว่า การหารเอาเศษเนี่ย มีประโยชน์มากๆ ในการเขียนโปรแกรม เช่น เอาไว้เช็คเลขคู่เลขคี่นั่นเองครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อ:** เครื่องหมายคณิตศาสตร์ (Arithmetic Operators)
- **ตารางเครื่องหมาย:**
  | เครื่องหมาย | ชื่อเรียก | หน้าที่ | ตัวอย่าง | ผลลัพธ์ |
  |---|---|---|---|---|
  | `+` | Plus | บวก | `5 + 3` | `8` |
  | `-` | Minus | ลบ | `10 - 4` | `6` |
  | `*` | Asterisk | คูณ | `4 * 2` | `8` |
  | `/` | Slash | หาร | `10 / 2` | `5` |
  | `%` | Modulo | หารเอาเศษ | `10 % 3` | `1` |
- **ภาพประกอบ:** รูปลูกอม 10 เม็ด แบ่งให้เด็ก 3 คน แล้วเหลือเศษ 1 เม็ด โยงไปที่เครื่องหมาย `%`

**💡 เทคนิคการสอน:**
- เน้นย้ำเรื่องเครื่องหมาย `*` และ `/` ว่าต้องใช้สัญลักษณ์นี้ในคีย์บอร์ด ห้ามพิมพ์ตัว x (เอ็กซ์)
- ถามคำถามทดสอบความเข้าใจเรื่อง Modulo สดๆ ในห้อง เช่น "แล้ว 15 % 4 ได้เท่าไหร่?" (ให้เด็กคิด 15 หาร 4 ได้ 3 เศษ 3 คำตอบคือ 3) หรือ "แล้ว 10 % 2 ล่ะ?" (หารลงตัว เศษคือ 0)

### 📄 หน้าที่ 32: วิธีการใช้งาน Operator + - * / %

**🎤 บทพูด (สิ่งที่ต้องพูด):**
"ทีนี้เรามาดูวิธีการเอาเครื่องหมายพวกนี้ไปเขียนในโค้ดจริงๆ กันครับ"
"หลักการทำงานของคอมพิวเตอร์เวลาเจอสมการคณิตศาสตร์ มันจะ **'คิดฝั่งขวาของเครื่องหมายเท่ากับ (=) ก่อนเสมอ'** แล้วค่อยเอาผลลัพธ์ไปเก็บไว้ในตัวแปรฝั่งซ้ายครับ"
"ดูจากโค้ดตัวอย่างบนกระดานนะครับ:"
"บรรทัดแรก พี่สร้างตัวแปร `int a = 5;` หมายความว่า สร้างกล่องชื่อ a ใส่เลข 5 ลงไป"
"บรรทัดต่อมา `int b;` สร้างกล่องชื่อ b เปล่าๆ เตรียมไว้"
"บรรทัดที่สาม `b = a + 5;` บรรทัดนี้แหละสำคัญ! คอมพิวเตอร์จะมองไปที่ฝั่งขวาก่อน คือ `a + 5` มันจะไปเปิดดูในกล่อง a อ้าว มีเลข 5 นี่นา มันก็เลยเอา 5 + 5 ได้ 10"
"พอได้ 10 ปุ๊บ มันก็จะเอาเลข 10 เนี้ย โยนเก็บลงไปในกล่อง b ทางซ้ายมือครับ!"
"และบรรทัดสุดท้าย `printf` เราก็สั่งให้มันโชว์ค่า b ออกมาบนหน้าจอ ซึ่งก็จะพิมพ์คำว่า `b = 10` นั่นเองครับ"
"เห็นไหมครับว่า การเขียนโปรแกรมไม่ได้ยากเลย มันทำตามลำดับขั้นตอนที่เราสั่งแบบเป๊ะๆ เลยครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อ:** วิธีการใช้งาน Operators
- **ตัวอย่างโค้ดพร้อมคำอธิบายแบบ Animation ทีละบรรทัด (ถ้าเป็นไปได้):**
  ```c
  int a = 5;       // สร้างตัวแปร a เก็บค่า 5
  int b;           // สร้างตัวแปร b รอไว้
  b = a + 5;       // คำนวณ a+5 (คือ 5+5=10) แล้วเอา 10 ไปเก็บใน b
  printf("b = %d", b);  // แสดงผลค่า b
  ```
- **ภาพประกอบ (Flow การคิด):**
  ลูกศรชี้จาก `a + 5` → คิดผลลัพธ์เป็น `10` → ลูกศรชี้โยงกลับไปเก็บที่ `b` (เน้นการทำงานขวาไปซ้ายของการกำหนดค่า)

**💡 เทคนิคการสอน:**
- อธิบายเรื่องเครื่องหมาย `=` (Assignment Operator) ให้ชัดเจน ว่าในภาษา C มันคือ "การกำหนดค่า" ไม่ใช่ "สมการความเท่ากัน" แบบในวิชาคณิตศาสตร์!
- ย้ำเสมอว่า "คอมพิวเตอร์ทำฝั่งขวาให้เสร็จก่อน แล้วค่อยโยนใส่ฝั่งซ้าย"
- ลองให้โจทย์สั้นๆ: `int x = 2; x = x + 3; printf("%d", x);` ถามว่า x จะออกมาเป็นอะไร? (ให้เด็กฝึกคิดว่า x เดิมคือ 2, เอา 2+3 ได้ 5, แล้วเอา 5 ไปทับค่าเก่าของ x)

### 📄 หน้าที่ 33: Title - คำสั่งควบคุมและคำสั่งวนซ้ำ (Flow Control & Loops)

**🎤 บทพูด (สิ่งที่ต้องพูด):**
"เอาล่ะครับ! ตอนนี้เราเก่งเรื่องการคำนวณละ แต่เดี๋ยวก่อน... โปรแกรมที่เราเขียนมาตั้งแต่ต้นเนี่ย มันยังทำงานแบบ **'ทื่อๆ'** อยู่เลยจริงไหมครับ?"
"คือมันทำงานจากบรรทัดบนสุด ลงมาล่างสุด ทำมันซะทุกบรรทัด ไม่เคยข้ามบรรทัดไหนเลย"
"แต่ในชีวิตจริงของพวกเรา มันไม่ได้เป็นเส้นตรงแบบนั้นใช่ไหมครับ? เราต้อง **ตัดสินใจ** ตลอดเวลาเลย!"
"เช่น ตื่นเช้ามา ถ้าฝนตก เราก็หยิบร่ม ถ้าไม่ตก เราก็ไม่หยิบ... หรือตอนเที่ยง ถ้าหิวมาก เราไปกินชาบู ถ้าหิวน้อย เราต้มมาม่ากิน"
"ชีวิตเรามันเต็มไปด้วย **'ทางแยก'** ครับ และบางเรื่องเราก็ทำซ้ำๆ เป็นลูปวงกลมด้วย"
"ดังนั้น ถ้าเราอยากให้โปรแกรมของเราฉลาดขึ้น คิดเองได้ ตัดสินใจได้ เราต้องรู้จักกับสิ่งนี้ครับ: **Flow Control (คำสั่งควบคุม) และ Loops (คำสั่งวนซ้ำ)**"
"นี่คือหัวใจสำคัญของการเขียนโปรแกรมเลยนะครับ พร้อมจะไปลุยกันหรือยัง?"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อใหญ่:** คำสั่งควบคุมและคำสั่งวนซ้ำ (Flow Control & Loops)
- **ภาพประกอบ:** รูปทางแยก (ซ้าย-ขวา) และรูปลูกศรวนเป็นวงกลม (รีไซเคิล หรือ วงเวียน)
- **ข้อความทิ้งท้าย:** "ทำให้โปรแกรมของคุณคิดและตัดสินใจได้!"

**💡 เทคนิคการสอน:**
- ใช้การเปรียบเทียบกับชีวิตประจำวันเยอะๆ (เช่น เรื่องฝนตก เรื่องของกิน) เพื่อให้เด็กๆ เห็นภาพว่าเงื่อนไข (Condition) คืออะไรก่อนที่จะไปดูโค้ด
- สร้างความตื่นเต้นว่านี่คือส่วนที่สนุกที่สุดของการเขียนโปรแกรม เพราะเรากำลังจะสอนให้คอมพิวเตอร์มีความคิด

### 📄 หน้าที่ 34: คำสั่งควบคุม (Flow Control) — if / else if / else

**🎤 บทพูด (สิ่งที่ต้องพูด):**
"มาดูตัวแรกกันครับ **การสร้างทางแยก** ในภาษา C เราทำให้คอมพิวเตอร์ 'เลือกทำ' ได้ด้วยคำสั่ง **`if - else if - else`** ครับ!"
"คำว่า `if` แปลว่า 'ถ้า' ... หลักการมันตรงตัวเลยครับ:"
"รูปแบบมันเป็นแบบนี้ครับ ดูโครงสร้างหลักๆ ก่อนนะ:"
"เราเริ่มด้วย `if` (ถ้า) แล้วตามด้วย (วงเล็บเงื่อนไข) ... ถ้าเงื่อนไขในวงเล็บเป็น **จริง (True)** มันก็จะเข้าไปทำงานในปีกกา `{ ... }` ทันทีครับ"
"แต่ถ้าเงื่อนไขแรกมันไม่จริงล่ะ? มันจะตกมาที่ทางเลือกถัดไป คือ `else if` (หรือว่าถ้า...) แล้วก็เช็คเงื่อนไขที่ 2"
"แล้วถ้าไม่มีอะไรเป็นจริงเลยล่ะ? สุดท้ายมันจะตกลงมาที่ `else` (มิฉะนั้น) ซึ่งเปรียบเสมือนตะกร้าใบสุดท้าย รองรับทุกอย่างที่ไม่เข้าเงื่อนไขด้านบนเลยครับ"
"ลองจินตนาการง่ายๆ ว่าเราเป็น Security (ยาม) ยืนอยู่หน้าประตูผับนะครับ เราต้องเช็คอายุคนก่อนเข้า:"
"ถ้าคนนั้นอายุ 20 ปีขึ้นไป (`if (age >= 20)`) → ปล่อยเข้าเลย! VIP!"
"ถ้าไม่ถึง แต่มีผู้ปกครองมาด้วย (`else if (has_parent == 1)`) → อนุโลมให้เข้าโซนเด็ก"
"แต่ถ้าไม่ใช่ทั้งสองอย่าง (`else`) → ไล่กลับบ้านไปนอนกินนมอุ่นๆ ซะ!"
"เห็นไหมครับ นี่แหละคือการทำงานของ if-else ! มันเช็คไปทีละด่านครับ"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อ:** คำสั่งควบคุม — `if / else if / else`
- **โครงสร้าง if-else (Syntax):**
  ```c
  if (เงื่อนไข 1) {
      // ทำส่วนนี้ ถ้าเงื่อนไข 1 เป็นจริง
  } else if (เงื่อนไข 2) {
      // ทำส่วนนี้ ถ้าเงื่อนไขแรกเท็จ แต่เงื่อนไข 2 เป็นจริง
  } else {
      // ทำส่วนนี้ ถ้าไม่มีเงื่อนไขไหนเป็นจริงเลย
  }
  ```
- **ภาพประกอบ (Flowchart):** แผนภูมิวงผังงาน (Flowchart) รูปข้าวหลามตัด (Decision) แยกซ้ายขวา (True/False) เพื่อให้เห็นภาพการไหลของโปรแกรมแบบมีทางแยก

**💡 เทคนิคการสอน:**
- เน้นการออกเสียงและแปลความหมายให้เด็กจำได้: if = ถ้า, else if = หรือถ้า, else = นอกเหนือจากนั้น
- ย้ำว่า `else if` จะมีกี่อันก็ได้ (มีหลายทางแยก) แต่ `if` ต้องมีอันเดียวตอนเริ่ม และ `else` มีอันเดียวตอนจบ (หรือไม่มี `else` เลยก็ได้)

### 📄 หน้าที่ 36: Operators สำหรับเปรียบเทียบเงื่อนไข และ การทำงานของ if-else

**🎤 บทพูด (สิ่งที่ต้องพูด):**
"แล้วเวลาเราจะเขียนเงื่อนไขในวงเล็บเนี่ย เราใช้อะไรเปรียบเทียบ? เราใช้ **'เครื่องหมายเปรียบเทียบ' (Comparison Operators)** ครับ"
"ดูบนจอเลย มีตัวที่คุ้นๆ เยอะ: `>` มากกว่า, `<` น้อยกว่า, `>=` มากกว่าหรือเท่ากับ, `<=` น้อยกว่าหรือเท่ากับ"
"แต่พี่ขอเตือนเลยนะ ไฮไลท์ไว้เลย! ดอกจัน 800 ล้านดวง! **เครื่องหมายเท่ากับ เราใช้ `==` (เท่ากับสองตัวติดกัน) นะครับ!**"
"จำสไลด์ก่อนหน้าที่พี่บอกได้ไหม? `=` ตัวเดียวแปลว่า 'เอาไปใส่' (การกำหนดค่า) แต่ถ้าจะ 'เปรียบเทียบ' ว่ามันเท่ากันไหม ต้องใช้ `==` เท่านั้น! ห้ามพลาดเด็ดขาด!"
"และถ้าจะบอกว่า 'ไม่เท่ากับ' เราใช้ `!=` (เครื่องหมายตกใจแล้วตามด้วยเท่ากับ) ครับ"
และ &&, หรือ || เพื่อเชื่อมการเปรียบเทียบหลายๆส่วน

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อ:** เครื่องหมายเปรียบเทียบ และ ตัวอย่างโค้ด
- **ตาราง Comparison Operators:**
  | เครื่องหมาย | ความหมาย | ตัวอย่าง |
  |---|---|---|
  | `==` | เท่ากับ (เปรียบเทียบ) | `x == 10` |
  | `!=` | ไม่เท่ากับ | `x != 0` |
  | `>` | มากกว่า | `score > 80` |
  | `<` | น้อยกว่า | `age < 18` |
  | `>=` | มากกว่าหรือเท่ากับ | `money >= 100` |
  | `<=` | น้อยกว่าหรือเท่ากับ | `temp <= 0` |
- **กล่องเตือนสีแดง (ตัวใหญ่ๆ):** ⚠️ ระวัง! `=` (กำหนดค่า) ไม่เหมือนกับ `==` (เปรียบเทียบ) ห้ามสับสน!

### 📄 หน้าที่ 37: Operators สำหรับเปรียบเทียบเงื่อนไข และ การทำงานของ if-else ตัวอย่าง

"เอาล่ะ มาดูโค้ดจริงกัน ตัวอย่างนี้เรื่องปากท้อง ทุกคนน่าจะเข้าใจง่ายสุด!"
"ดูที่โค้ดนะครับ สมมติพี่มีเงิน `int money = 50;`"
"โปรแกรมเริ่มเช็คบรรทัดแรก: `if (money >= 100)` — พี่ถามพวกเราหน่อย 50 มากกว่าหรือเท่ากับ 100 ไหม? (เด็กตอบ: ไม่!)"
"ใช่ครับ ไม่จริง! (False) ดังนั้นโปรแกรมจะไม่สนปีกกาของ if เลย มันจะกระโดดข้ามไปด่านต่อไปทันที!"
"ด่านต่อไปคือ `else if (money >= 50)` — ถามว่า 50 มากกว่าหรือเท่ากับ 50 ไหม? (เด็กตอบ: จริง!)"
"ใช่ครับ! พอเป็นจริง (True) ปุ๊บ โปรแกรมจะมุดเข้าไปในปีกกานั้นทันที แล้วสั่งปริ้นท์คำว่า **'กินกะเพรา'**"

"ดังนั้น กฎเหล็กของ if-else คือ **'ลำดับสำคัญมาก'** โปรแกรมเช็คจากบนลงล่าง เจออันไหนจริงก่อน ทำอันนั้น แล้วจบเลย!"
"เหมือนตะแกรงร่อนทรายครับ เราต้องเอาเงื่อนไขที่แคบที่สุด รูเล็กที่สุดไว้บนสุด ไม่งั้นมันจะร่วงทะลุหมด!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**

- **ตัวอย่างโค้ด:**
  ```c
  int money = 50;

  if (money >= 100) {
      printf("กินชาบู\n");
  } else if (money >= 50) {
      printf("กินกะเพรา\n");
  ```

  *(เฉลย: พิมพ์ 'กินกะเพรา' ทั้งที่พี่รวยพอจะกินชาบู! เพราะ 150 >= 50 เป็นจริงด่านแรก มันเลยทำงานแล้วจบเลย นี่คือความสำคัญของการเรียงลำดับ!)*

### 📄 หน้าที่ 39: คำสั่งวนซ้ำ (Loops) — While Loop

"ในภาษา C มี Loop หลักๆ 2 แบบที่เราต้องรู้จัก วันนี้เรามาดูตัวแรกก่อน คือ **While Loop**"
"คำว่า `while` แปลว่า 'ตราบเท่าที่...' ... กฎของมันคือ: **วนไปเรื่อยๆ ตราบเท่าที่เงื่อนไขยังเป็นจริง!**"
"เรามักจะใช้ While Loop ในสถานการณ์ที่ **เราไม่รู้จำนวนรอบที่แน่นอน** ครับ เช่น 'ให้ผู้ใช้พิมพ์รหัสผ่านไปเรื่อยๆ จนกว่าจะถูก' เราไม่มีทางรู้เลยว่าเขาจะพิมพ์ผิดกี่ครั้ง อาจจะครั้งเดียว หรือ 10 ครั้ง"

"ดูตัวอย่างโค้ดนะครับ:"
"เราเริ่มต้นตั้งค่าให้ `int i = 0;` (ตัวแปรนับรอบ)"
"บรรทัด `while (i < 5)` คอมจะถามตัวเองว่า 'ตอนนี้ i น้อยกว่า 5 ใช่ไหม?'"
"รอบแรก i เป็น 0... ใช่! (True) มันก็มุดเข้าไปปริ้นท์เลข 0 ออกมา"
"และ **บรรทัดที่สำคัญระดับชีวิต `i++;`** บรรทัดนี้แปลว่า ให้เอา i ไปบวกเพิ่มอีก 1 (ตอนนี้ i จะกลายเป็น 1 แล้วนะ)"
"พอมันทำงานจบปีกกา มันจะกระโดดกลับขึ้นไปเช็คเงื่อนไขใหม่ครับ: '1 < 5 ใช่ไหม?' -> ใช่! -> ปริ้นท์ 1 -> i กลายเป็น 2"
"มันจะวนแบบนี้ไปเรื่อยๆ... จนกระทั่งรอบนึง i ถูกบวกจนกลายเป็น 5... พอมันเด้งกลับไปเช็คเงื่อนไข '5 < 5 ใช่ไหม?' -> คราวนี้ **ไม่จริง! (False)** แล้วครับ!"
"พอเงื่อนไขเป็น False ปุ๊บ วงแหวนจะแตกทันที! โปรแกรมจะเด้งออกจาก Loop แล้วไปทำคำสั่งข้างล่างต่อ จบการวนซ้ำครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อ:** คำสั่งวนซ้ำ (Loops) — While Loop (ไม่รู้จำนวนรอบที่แน่นอน)
- **ภาพประกอบ:** ภาพวงเวียนจราจร หรือภาพลู่วิ่ง
- **โครงสร้าง syntax:**
  ```c
  while (เงื่อนไข) {
      // คำสั่งที่จะให้ทำซ้ำ
      // ⚠️ ต้องมีบรรทัดที่ทำให้เงื่อนไขเปลี่ยนเป็น False ได้ด้วย!
  }
  ```
- **ตัวอย่างโค้ด:**
  ```c
  int i = 0;
  while (i < 5) {
      printf("%d\n", i);
      i++;   // สำคัญมาก! คือการบวกค่า i ทีละ 1 (i = i + 1)
  }
  ```
- **กล่องเตือน (Infinite Loop):**
  ⚠️ **ระวัง!** ถ้าลืมใส่ `i++;` ค่า i จะเป็น 0 ตลอดไป ทำให้ `0 < 5` เป็นจริงตลอดกาล โปรแกรมจะวนไม่จบสิ้น (Infinite Loop) จนกว่าคอมจะค้าง!

**💡 เทคนิคการสอน:**
- **Trace โค้ดให้ดู:** เขียนค่า i ไว้มุมกระดาน แล้วทดเลขให้ดูทีละรอบ (0, 1, 2, 3, 4, 5 -> ออกลูป) เพื่อให้เด็กเห็นภาพการเปลี่ยนแปลงค่าในเมมโมรี่
- **Live Demo (เรียกเสียงฮือฮา):** เปิดโปรแกรมเขียนโค้ดสดๆ แล้วจงใจลบคำสั่ง `i++;` ออก กดรันให้เด็กดูว่าคอมพิวเตอร์มันพ่นเลข 0 รัวๆ ไม่หยุดจนโปรแกรมค้างยังไง (เรียกว่า Infinite loop เป็นเรื่องที่โปรแกรมเมอร์ทุกคนต้องเคยเจอ!)

### 📄 หน้าที่ 40: คำสั่งวนซ้ำ (Loops) — For Loop

**🎤 บทพูด (สิ่งที่ต้องพูด):**
"มาดู Loop ตัวที่ 2 ครับ ชื่อว่า **For Loop**"
"ถ้าย้อนกลับไปดู while loop เมื่อกี้ น้องๆ จะเห็นว่าเราต้องเขียนส่วนประกอบ 3 ที่แยกกัน คือ: 1. ตั้งค่าเริ่มต้น (`int i = 0;` ไว้ข้างนอก) 2. เงื่อนไข (`i < 5` ไว้ในวงเล็บ) และ 3. การเปลี่ยนค่า (`i++;` ไว้ข้างใน)"
"แต่สําหรับ For Loop... มันออกแบบมาให้ **รวบตึงทั้ง 3 อย่างนี้ไว้ในวงเล็บบรรทัดเดียวเลย!** ทำให้โค้ดเราสั้น เป็นระเบียบ และป้องกันการลืมเปลี่ยนค่าจนเกิด Infinite Loop ครับ"

"เราจะใช้ For Loop ตอนไหน? เราจะใช้ตอนที่ **เรารู้จำนวนรอบที่ชัดเจนแน่นอน** ครับ เช่น สั่งให้ปริ้นท์ 1 ถึง 100, หรือสั่งวนนับถอยหลัง 10 วินาที"

"ดูตัวอย่างนาฬิกานับถอยหลังบนจอนะครับ:"
"ในวงเล็บ `for (int i = 5; i > 0; i--)` เราแบ่งเป็น 3 ท่อนคั่นด้วยเครื่องหมายเซมิโคลอน `;`"
"1. **จุดเริ่มต้น:** `int i = 5;` — ให้ i เริ่มที่ 5 (ทำแค่รอบแรกสุดรอบเดียว)"
"2. **เงื่อนไข:** `i > 0;` — จะให้วนตราบใดที่ i ยังมากกว่า 0"
"3. **การเปลี่ยนค่า:** `i--` — ทุกครั้งที่วนเสร็จรอบนึง ให้ลดค่า i ลงทีละ 1 (ตรงข้ามกับ ++ นะครับ)"

"รอบแรก i=5... 5>0 จริง... ปริ้นท์ 'เหลือเวลา 5 วินาที'"
"จากนั้นมันจะวิ่งไปทำ `i--` กลายเป็น 4... แล้วเช็คใหม่ 4>0 จริง... ปริ้นท์ 'เหลือเวลา 4 วินาที'"
"ทำไปเรื่อยๆ จนปริ้นท์ 1... แล้ว i ลดลงเหลือ 0"
"พอเช็คเงื่อนไข '0 > 0' → **เท็จ! (False)** → ออกจากลูปทันที! แล้วมาปริ้นท์ 'หมดเวลา!' ข้างนอกครับ"
"นี่แหละครับ For Loop เขียนบรรทัดเดียว จบ ครบ กระชับสุดๆ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อ:** คำสั่งวนซ้ำ (Loops) — For Loop (รู้จำนวนรอบชัดเจน)
- **โครงสร้าง syntax:**
  ```c
  for (จุดเริ่มต้น; เงื่อนไข; การเปลี่ยนค่า) {
      // คำสั่งที่จะให้ทำซ้ำ
  }
  ```
- **ตัวอย่างโค้ด:**
  ```c
  for (int i = 5; i > 0; i--) {
      printf("เหลือเวลา %d วินาที\n", i);
  }
  printf("หมดเวลา!\n");
  ```
- **ตารางเปรียบเทียบสั้นๆ (While vs For):**
  - **While Loop:** ใช้เมื่อ *ไม่รู้* จำนวนรอบที่แน่นอน (รอผู้ใช้พิมพ์ถูก)
  - **For Loop:** ใช้เมื่อ *รู้* จำนวนรอบล่วงหน้าชัดเจน (เช่น วน 10 รอบ)

**💡 เทคนิคการสอน:**
- เน้นให้เด็กลองสังเกตว่า `i++` คือเพิ่มทีละ 1 ส่วน `i--` คือลดทีละ 1
- ให้โจทย์สด: "ถ้าน้องอยากให้มันปริ้นท์คำว่า 'ไอเลิฟซี' 3 รอบ น้องจะแก้โค้ด For loop อย่างไร?" (ให้เด็กคิดจุดเริ่ม และเงื่อนไขจบ เช่น `for(int i=1; i<=3; i++)`)
- สรุปความเหมือนและต่างของ While กับ For ให้เคลียร์ ว่ามันทำซ้ำได้เหมือนกันเป๊ะ แค่จัดระเบียบโค้ดต่างกันเฉยๆ

### 🏋️ แบบฝึกหัด 3.1: เครื่องตัดเกรด (Grade Calculator)

**🎤 บทพูด (สิ่งที่ต้องพูด):**

"มาถึงข้อคลาสสิกตลอดกาลของการเรียนเขียนโปรแกรมครับ โปรแกรมตัดเกรด! ข้อนี้เราจะใช้ท่าไม้ตาย `if-else if` มาช่วยครับ ก่อนจะพิมพ์โค้ด เรามาวิเคราะห์ลอจิกกันอย่างละเอียดก่อนดีกว่า"

"กฎเหล็กของการใช้ `if-else if` ซ้อนๆ กันก็คือ — **ลำดับการวางเงื่อนไข (Order of Conditions) สำคัญมากถึงมากที่สุดครับ!** ยังจำเรื่องตะแกรงร่อนทรายที่พี่เคยสอนได้ไหม? น้ำจะไหลผ่านตะแกรงชั้นแรกไปชั้นที่สอง ก็ต่อเมื่อมันไม่ถูกตะแกรงชั้นแรกดักไว้เท่านั้น!"

"ลองจินตนาการเล่นๆ นะครับ ถ้าเราเรียงโค้ดจากเงื่อนไขต่ำสุดไปสูงสุด โดยเอา `score >= 50` ขึ้นไปไว้บรรทัดแรกสุดของ if เลย แล้วมีคนสอบได้ 100 คะแนนเต็มมา (ซึ่งควรจะได้เกรด A ถูกไหม?) แต่พอโปรแกรมเช็คตะแกรงแรก `100 >= 50 ไหม?` ปรากฏว่า 'จริง!' โปรแกรมก็จะพิมพ์ว่าคุณได้เกรด D ทันที แล้วก็กระโดดข้ามเงื่อนไขอื่นๆ ไปหมดเลย! จบกัน เกรด A ของฉัน!"

"ดังนั้น เพื่อป้องกันปัญหานี้ เราต้อง **เรียงเงื่อนไขจากแคบที่สุด (หรือเงื่อนไขที่เกิดยากที่สุด) ไปหาเงื่อนไขที่กว้างที่สุด** ครับ:"
"1. เช็ค Cheating (`> 100`) ก่อนเลยครับ นี่คือเงื่อนไขดักจับคนโกงที่แคบที่สุด"
"2. ถ้าไม่โกง ค่อยมาเช็คเกรด A (`>= 80`)"
"3. ถ้าไม่ถึง A ค่อยเช็คเกรด B (`>= 70`)"
"4. เช็คเกรด C (`>= 60`)"
"5. เช็คเกรด D (`>= 50`)"
"6. ท้ายที่สุด ถ้าไม่ตรงกับอะไรข้างบนเลย ก็ตกไปอยู่ที่ตะแกรงสุดท้าย คือ `else` → เกรด F นั่นเอง!"

"มาดูโค้ดกันครับ:"
```c
#include <stdio.h>

int main() {
    int score;
    printf("Enter score: ");
    scanf("%d", &score);

    if (score > 100) {
        printf("Cheating!\n");
    } else if (score >= 80) {
        printf("Grade A\n");
    } else if (score >= 70) {
        printf("Grade B\n");
    } else if (score >= 60) {
        printf("Grade C\n");
    } else if (score >= 50) {
        printf("Grade D\n");
    } else {
        printf("Grade F\n");
    }
    return 0;
}
```

"เรามาลอง Trace โค้ด (ไล่การทำงานทีละบรรทัด) กันดูนะครับ เพื่อความชัวร์"
"**เคสที่ 1: ถ้าผู้ใช้ป้อนคะแนน 75**"
"- `75 > 100` จริงไหม? **ไม่จริง** → กระโดดข้าม"
"- `75 >= 80` จริงไหม? **ไม่จริง** → กระโดดข้าม"
"- `75 >= 70` จริงไหม? **จริง!** → คอมพิวเตอร์จะพิมพ์ 'Grade B' ออกมา แล้วกระโดดออกจากกล่อง if ทันที ไม่ไปเช็ค C D F ต่อแล้ว! (ทำงานถูกต้อง!)"

"**เคสที่ 2: ถ้าผู้ใช้ป้อนคะแนน 45**"
"- เช็คไปเรื่อยๆ จนถึง `>= 50` ก็ยังไม่ผ่าน → สุดท้ายคอมพิวเตอร์จะตกไปอยู่ในโซน `else` และพิมพ์ 'Grade F' ออกมา (ทำงานถูกต้อง!)"

"**เคสที่ 3: ถ้ามีคนแอบโกง ป้อนคะแนน 120**"
"- เข้าเงื่อนไขแรกปุ๊บ `120 > 100`? **จริง!** → พิมพ์ 'Cheating!' ทันที แล้วหยุดทำงาน ไม่ไปเช็คเกรด A! (จับคนโกงได้สำเร็จ!)"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **ตารางเงื่อนไขเกรด:** แสดงคะแนนและเกรดที่สอดคล้องกันเป็นตาราง
- **โค้ดเต็ม** ของการใช้ `if-else if` ต่อกันหลายๆ ชั้น
- **Trace Table จำลอง:** แสดงการไล่โค้ดทีละบรรทัดของคะแนน 75, 45, และ 101 เพื่อแสดงให้เห็นว่าการเรียงลำดับมีผลอย่างไร
- **กล่อง Key Insight (ข้อควรระวังสำคัญ):** ⚠️ ต้องเรียงเงื่อนไขจากแคบไปกว้าง (หรือจากเกณฑ์สูงไปต่ำ) เสมอ! และเงื่อนไขพิเศษ (Cheating) ต้องอยู่บนสุด!

**💡 เทคนิคการสอน:**
- **เรียนรู้จากข้อผิดพลาด (Learn by breaking things):** ท้าให้เด็กลองสลับลำดับโค้ดในเครื่องตัวเอง โดยเอา `else if (score >= 50)` ขึ้นไปวางไว้บนสุด แล้วลองรันโปรแกรมโดยใส่คะแนน 95 ลงไป ให้เด็กเห็นกับตาตัวเองว่ามันได้เกรด D ออกมาจริงๆ! เด็กจะจำหลักการเรียงลำดับนี้ไปตลอดชีวิต
- **ทำ Trace Table บนกระดาน:** การไล่โค้ดบนกระดานทีละบรรทัด เป็นวิธีที่ดีที่สุดในการทำให้เด็กเข้าใจการทำงานแบบล่างลงบนของคอมพิวเตอร์
- **Challenge:** ท้าทายให้เด็กเพิ่มเงื่อนไข `ถ้าคะแนนติดลบ (< 0)` ให้พิมพ์คำว่า `Invalid Score!` ลงไปในโปรแกรมด้วยตัวเอง

### 🏋️ แบบฝึกหัด 3.2: ตรวจสอบเลขคู่/คี่ (Even/Odd Checker)

**📝 โจทย์:** รับตัวเลขจำนวนเต็ม 1 ตัว แล้วบอกว่าเป็นเลขคู่ (Even) หรือเลขคี่ (Odd)

**🎤 บทพูด (สิ่งที่ต้องพูด):**

"ข้อนี้ดูเหมือนเป็นโจทย์คณิตศาสตร์ธรรมดาๆ แต่มันจะสอนเราให้รู้จักกับตัวดำเนินการ (Operator) ที่สำคัญและมีประโยชน์มากที่สุดตัวหนึ่งในการเขียนโปรแกรม นั่นคือ **เครื่องหมาย % (Modulus)** ครับ หรือที่โปรแกรมเมอร์เรียกกันสั้นๆ ว่า 'ม็อด (Mod)' — ซึ่งมันคือ 'การหารเพื่อเอาเศษ' ครับ ห้ามสับสนกับเครื่องหมายเปอร์เซ็นต์ในวิชาเลขเด็ดขาดนะ!"

"ก่อนอื่น เรากลับมาที่คำถามคลาสสิก ใครจำได้บ้างว่าในทางคณิตศาสตร์ เลขคู่ (Even) นิยามของมันคืออะไร?"
(รอเด็กตอบ... "เลขที่หาร 2 ลงตัว!")

"ถูกต้องที่สุดครับ! เลขคู่คือเลขที่เมื่อเอามาหาร 2 แล้ว มัน 'เหลือเศษ 0' ครับ เช่น 4 ÷ 2 = 2 เหลือเศษ 0, หรือ 10 ÷ 2 = 5 เหลือเศษ 0"
"ส่วนเลขคี่ (Odd) คือเลขที่หาร 2 แล้ว 'เหลือเศษ 1' เสมอครับ เช่น 5 ÷ 2 = 2 เหลือเศษ 1, หรือ 7 ÷ 2 = 3 เหลือเศษ 1"

"ในภาษา C เราใช้เครื่องหมาย `%` เพื่อหา 'เศษ' จากการหารครับ ดังนั้น:"
"- `10 % 2` (สิบม็อดสอง) คำตอบคือ 0 → แปลว่าเป็นเลขคู่"
"- `7 % 2` (เจ็ดม็อดสอง) คำตอบคือ 1 → แปลว่าเป็นเลขคี่"

"พอเรารู้แบบนี้ ลอจิกในการเขียนโปรแกรมก็ง่ายเหมือนปอกกล้วยเข้าปากเลยครับ:"
"```text
ถ้า (ตัวเลขนั้นถูก % ด้วย 2 แล้วผลลัพธ์มันมีค่าเท่ากับ 0 พอดี)
    แสดงว่ามันเป็นเลขคู่!
ถ้าไม่ใช่แบบนั้น (else)
    มันก็ต้องเป็นเลขคี่แน่นอน!

"มาดูโค้ดกันครับ:"
```c
#include <stdio.h>

int main() {
    int num;
    printf("Enter a number: ");
    scanf("%d", &num);

    // เช็คเงื่อนไขหารเอาเศษด้วย 2 แล้วเท่ากับ 0 หรือไม่
    if (num % 2 == 0) {
        printf("%d is Even (เลขคู่)\n", num);
    } else {
        printf("%d is Odd (เลขคี่)\n", num);
    }
    return 0;
}
```

"ข้อนี้เป็น Pattern (รูปแบบ) ที่สำคัญมากนะครับ การใช้ `% (Modulus)` ไม่ได้จบแค่เช็คเลขคู่เลขคี่ ในอนาคตเราจะใช้มันอีกเยอะมาก! เช่น เช็คว่าเลขนี้หาร 3 ลงตัวไหม? เช็คว่าปีนี้เป็นปีอธิกสุรทิน (Leap Year) หรือเปล่าโดยดูว่าหาร 4 ลงตัวไหม? หรือแม้แต่เอาไปประยุกต์ทำระบบสุ่ม Lucky Draw ในเวิร์คช็อปที่เราจะได้ทำกันในวันสุดท้ายด้วยครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หลักการคณิตศาสตร์:** ทบทวนว่า เลขคู่ = หาร 2 เหลือเศษ 0 / เลขคี่ = หาร 2 เหลือเศษ 1
- **อธิบายเครื่องหมาย `%`:** Modulus = "การหารเอาเศษ" (ไม่ใช่ร้อยละหรือเปอร์เซ็นต์!)
- **ตัวอย่างการคิดในคอมพิวเตอร์:** แสดง `10 % 2 = 0` (Even), `7 % 2 = 1` (Odd)
- **โค้ดเต็ม + Output การทดสอบโปรแกรม**

**💡 เทคนิคการสอน:**
- **ซักซ้อมความเข้าใจ:** ตั้งโจทย์เลขง่ายๆ ให้เด็กคำนวณ 'ม็อด' ในใจก่อนเริ่มเขียนโค้ด เช่น `15 % 2 = ?` (ตอบ 1), `20 % 3 = ?` (ตอบ 2 เศษ), `100 % 7 = ?` (ตอบ 2) เพื่อเช็คว่าเด็กเข้าใจหลักการหารเอาเศษจริงๆ
- **Trick Question:** ถามเด็กๆ ว่า "แล้ว 0 ล่ะ? 0 เป็นเลขคู่หรือเลขคี่?" (เฉลยคือ เป็นเลขคู่! เพราะ 0 % 2 ก็ได้เศษ 0 เช่นกัน)
- **Challenge สุดท้าทาย:** "ใครเสร็จแล้ว ลองแก้โค้ดเปลี่ยนเป็นโปรแกรม 'เช็คว่าตัวเลขที่กรอกมา หารด้วย 3 ลงตัวหรือไม่' ให้พี่ดูหน่อย!" (เด็กแค่เปลี่ยนจาก % 2 เป็น % 3)

### 🏋️ แบบฝึกหัด 3.3: โปรแกรมสูตรคูณ (Multiplication Table)

**📝 โจทย์:** รับตัวเลขแม่สูตรคูณ 1 ตัว (เช่น แม่ 2) แล้วใช้ loop พิมพ์สูตรคูณแม่นั้นตั้งแต่ ×1 ไปจนถึง ×12 ออกมาทางหน้าจอ

**🎤 บทพูด (สิ่งที่ต้องพูด):**

"เอาล่ะครับ! ข้อนี้เป็นข้อแรกที่เราจะได้ดึงเอาพลังของ **For Loop** มาใช้งานแบบจริงๆ จังๆ สักที! เรามาเริ่มวิเคราะห์โจทย์กันเหมือนเดิมครับ"

"โจทย์ต้องการให้เราพิมพ์สูตรคูณ 12 บรรทัด (คูณ 1 ถึง คูณ 12) นี่เป็นคีย์เวิร์ดสำคัญเลยครับ! การที่เรา 'รู้จำนวนรอบที่จะต้องทำซ้ำอย่างแน่ชัด' ว่าต้องทำกี่รอบ (ข้อนี้คือ 12 รอบ) บ่งบอกว่าพระเอกของงานนี้คือ **For Loop** แน่นอนครับ!"

"มาลองวิเคราะห์บรรทัดที่จะโชว์บนหน้าจอกันครับ สมมติว่าผู้ใช้กรอกแม่สูตรคูณแม่ 2:"
```text
2 x 1 = 2
2 x 2 = 4
2 x 3 = 6
...
2 x 12 = 24
```

"น้องๆ ลองสังเกตดีๆ นะครับ ในแต่ละบรรทัดมีอะไรเหมือนเดิม และมีอะไรเปลี่ยนไปบ้าง?"
"1. ตัวเลขตัวแรกสุด (2) มัน **ไม่เปลี่ยนเลย** ในทุกๆ บรรทัด! นี่ก็คือตัวแปร `base` ที่เราจะรับมาจากคำสั่ง scanf นั่นเอง"
"2. ตัวเลขตัวที่สอง (1, 2, 3 ไปจนถึง 12) มัน **เปลี่ยนค่าเพิ่มขึ้นทีละ 1 ทุกบรรทัด**! เอ๊ะ... คุ้นๆ ไหมครับ? ตัวแปรอะไรน้าที่วิ่งขึ้นทีละ 1 ทุกรอบใน for loop? ใช่แล้วครับ! มันก็คือ **ตัวแปร `i`** (index/iterator) นั่นเอง!"

"ดังนั้น ภายในลูป 12 รอบ สิ่งที่เราต้องทำก็มีแค่บรรทัดเดียว คือการพิมพ์ข้อความโดยใช้ตัวแปร `base` และ `i` มาประกอบกัน:"
"`printf("%d x %d = %d\n", base, i, base * i);`"

"ง่ายและสวยงามมากเลยใช่ไหมครับ? `base` คงที่ตลอด, `i` เปลี่ยนค่าวิ่งไปเรื่อยๆ, ส่วนผลคูณ `base * i` คอมพิวเตอร์ก็คำนวณสดๆ ใหม่ๆ ใน printf ให้ทุกรอบเลย"

"เรามาดูโค้ดเต็มๆ กันครับ:"
```c
#include <stdio.h>

int main() {
    int base;
    printf("Enter number (แม่สูตรคูณ): ");
    scanf("%d", &base);

    // ให้ลูปวิ่งตั้งแต่ i=1 ไปจนถึง i=12
    for (int i = 1; i <= 12; i++) {
        printf("%d x %d = %d\n", base, i, base * i);
    }
    
    return 0;
}
```

"มาลอง Trace โปรแกรม (ไล่โค้ด) ดูสัก 3 รอบแรกกันครับ สมมติ `base = 3`:"
"- เข้าลูปรอบแรก `i = 1`: พิมพ์ `3 x 1 = 3` (เพราะ 3*1=3)"
"- เข้าลูปรอบสอง `i = 2`: พิมพ์ `3 x 2 = 6` (เพราะ 3*2=6)"
"- เข้าลูปรอบสาม `i = 3`: พิมพ์ `3 x 3 = 9`"
"- โปรแกรมก็จะวนทำแบบนี้ไปเรื่อยๆ จนถึงรอบสุดท้าย..."
"- เข้ารอบที่ 12 `i = 12`: พิมพ์ `3 x 12 = 36`"
"- พอจบรอบที่ 12, `i` จะถูกบวกเพิ่มกลายเป็น 13 เครื่องจะมาเช็คเงื่อนไข `13 <= 12` หรือเปล่า? คำตอบคือ 'เท็จ!' ลูปก็จะแตกทันที โปรแกรมจบการทำงานครับ"

"ข้อนี้เป็นจุดเปลี่ยนสำคัญเลยครับ! มันสอนให้เรารู้ว่า **ตัวแปร `i` ภายในลูปนั้น มันมีพลังมากกว่าแค่การเป็น 'ตัวนับรอบ' ธรรมดาๆ** เราสามารถเอาค่าของมันมาคำนวณ เอามาโชว์ที่หน้าจอ เอามาทำอะไรก็ได้ในแต่ละรอบ! นี่คือแก่นแท้ความเก่งกาจของ for loop ที่โปรแกรมเมอร์ทุกคนต้องเข้าใจครับ"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โจทย์:** รับแม่สูตรคูณ → ใช้ Loop พิมพ์ตั้งแต่ ×1 ถึง ×12
- **วิเคราะห์โครงสร้างข้อมูล (Pattern Matching):**
  - แสดงภาพสูตรคูณ ไฮไลท์สีตัวเลขแม่สูตรคูณว่า "ค่าคงที่ (`base`)"
  - ไฮไลท์สีตัวเลขตัวคูณ 1-12 ว่า "ค่าที่เปลี่ยนแปลง (`i`)"
- **โค้ดเต็ม + Output ตัวอย่าง:** แสดง Output เป็นตารางแม่สูตรคูณสวยงาม
- **Trace Table (สำคัญมาก):** สร้างตารางแสดงรอบการทำงาน (i = 1, 2, 3...) และค่าที่แสดงผลออกมาแต่ละรอบ เพื่อให้เห็นภาพการหมุนของ Loop ชัดเจน

**💡 เทคนิคการสอน:**
- **วาด Trace Table บนกระดาน:** เป็นการสอน For Loop ที่ได้ผลที่สุด ให้เด็กเห็นว่า `i` มันเปลี่ยนค่าในแต่ละรอบจริงๆ
- **Level Up:** "ใครเสร็จแล้ว ลองแก้นิดเดียว ให้มันท่องสูตรคูณไปถึงแม่ 24 หรือ 100 ดูสิ!" (เด็กจะว้าวว่าคอมพิวเตอร์ทำของเหนื่อยๆ แบบนี้ได้ในพริบตา)
- **Reverse Engineering Challenge:** ท้าทายคนเก่ง! "ลองเขียนลูปให้มันท่องสูตรคูณแบบ 'ถอยหลัง' จาก ×12 ลงมาจนถึง ×1 ดูสิครับ ใครทำได้บ้าง!" (ใบ้: ต้องเปลี่ยนการเริ่ม `i=12`, เช็ค `i>=1`, และสั่ง `i--`)

### 🏋️ แบบฝึกหัด 3.4: พิมพ์ดวงดาว (Print Stars)

**📝 โจทย์:** รับค่าตัวเลขเต็มบวก `N` จากผู้ใช้ แล้วพิมพ์สัญลักษณ์ดวงดาว `*` ติดต่อกันเป็นความยาว `N` ดวงในบรรทัดเดียวกัน

**🎤 บทพูด (สิ่งที่ต้องพูด):**

"ข้อนี้มองเผินๆ เหมือนจะง่ายมากครับ แต่จริงๆ แล้วมีเทคนิคเล็กๆ แต่สำคัญมากซ่อนอยู่ เป้าหมายของเราคือ ถ้าผู้ใช้ป้อนเลข 5 โปรแกรมจะต้องแสดงดาวติดกันแบบนี้ `*****` ออกมาบนจอครับ"

"ลอจิกพื้นฐานทุกคนน่าจะนึกออกแล้ว คือเราต้องใช้ `for loop` วนไปเลย `N` รอบ และในแต่ละรอบ เราก็สั่งให้คอมพิวเตอร์พิมพ์ดาว 1 ดวงออกมา"

"แต่เดี๋ยวก่อน! ที่ผ่านมาเวลาเราพิมพ์อะไรด้วยคำสั่ง `printf` เรามักจะใส่ `\n` (New Line) ต่อท้ายเสมอใช่ไหมครับ? สำหรับข้อนี้ **ห้ามใส่ `\n` ตอนพิมพ์ดาวเด็ดขาดเลยนะครับ!** ลองนึกภาพดู ถ้าเราใส่ `\n` ดาวมันก็จะหล่นลงมาทีละบรรทัด กลายเป็นแนวดิ่งแบบนี้แทน:"
```text
*
*
*
*
*
```
"ซึ่งมันไม่ใช่สิ่งที่เราต้องการ! เราต้องการดาวที่เรียงเป็นแถวแนวนอน `*****` ในบรรทัดเดียวกันครับ"

"ดังนั้น ภายในวงเล็บของ loop เราจะใช้แค่คำสั่ง `printf("*");` เพียวๆ (ไม่มี \n) ดาวก็จะถูกพิมพ์ต่อๆ กันไปในบรรทัดเดียวกันครับ แล้วพอ 'ลูปทำงานเสร็จสิ้นทั้งหมด' หรือออกจากลูปแล้ว เราค่อยใช้คำสั่ง `printf("\n");` เดี่ยวๆ สักบรรทัดนึง เพื่อให้เคอร์เซอร์มันลงไปบรรทัดใหม่ เป็นการจบแถวแบบสวยงามครับ"

"มาดูโค้ดกัน:"
```c
#include <stdio.h>

int main() {
    int n;
    printf("Enter N (จำนวนดาว): ");
    scanf("%d", &n);

    // วนลูป n รอบ พิมพ์ดาวทีละดวงต่อๆ กัน
    for (int i = 0; i < n; i++) {
        printf("*");  // ⚠️ ไม่มี \n!
    }
    
    // พอลูปจบแล้ว ค่อยพิมพ์ขึ้นบรรทัดใหม่
    printf("\n");  
    
    return 0;
}
```

"ลองไล่สเต็ปดูเล่นๆ สมมติผู้ใช้ป้อน `n = 5`:"
"- รอบที่ 1: พิมพ์ * → หน้าจอแสดง: `*`"
"- รอบที่ 2: พิมพ์ * → หน้าจอแสดง: `**` (ดาวดวงใหม่มาต่อท้ายดวงเก่า)"
"- รอบที่ 3: พิมพ์ * → หน้าจอแสดง: `***`"
"- รอบที่ 4: พิมพ์ * → หน้าจอแสดง: `****`"
"- รอบที่ 5: พิมพ์ * → หน้าจอแสดง: `*****`"
"- จบลูป! ออกมาเจอคำสั่ง `printf("\n");` → เคอร์เซอร์กระพริบลงไปบรรทัดถัดไป เป็นอันเสร็จสิ้น!"

"ข้อนี้ถึงจะสั้น แต่เป็น **พื้นฐานที่โคตรสำคัญสำหรับการไปทำเวิร์คช็อป Star Pyramid** ในอนาคตครับ! ถ้าเราเข้าใจการควบคุมบรรทัด (การใส่/ไม่ใส่ `\n`) ในข้อนี้ ข้อพีระมิดที่จะต้องใช้ลูปซ้อนลูปก็จะง่ายขึ้นเป็นกองเลยครับ"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โจทย์:** รับค่า N (เช่น N=5) → แสดงผลดวงดาวติดกัน `*****`
- **Key Insight (จุดตาย):** `printf("*");` ไม่มี `\n` = ไม่ขึ้นบรรทัดใหม่! พิมพ์ต่อกันไปเรื่อยๆ จนกว่าจะสั่ง \n แยกต่างหากนอกลูป
- **ภาพเปรียบเทียบผิด/ถูก:**
  - รูปแบบที่ถูก (ไม่มี \n ในลูป) → ดาวเรียงแนวนอน
  - รูปแบบที่ผิด (มี \n ในลูป) → ดาวเรียงแนวดิ่ง
- **โค้ดเต็ม + Output**

**💡 เทคนิคการสอน:**
- **ลองทำแบบผิดๆ ดูก่อน:** แนะนำให้เด็กแกล้งเขียนแบบมี `\n` ไปก่อน พอกดรันแล้วเห็นดาวร่วงลงมาเป็นสายฝน เด็กจะเข้าใจทันทีว่าอ๋อออ \n มันทำงานแบบนี้นี่เอง แล้วค่อยให้เด็กลบออก
- **Mini-Challenge สนุกๆ:** "ใครปริ้นท์ดาวเสร็จแล้ว ลองพิมพ์ตัวเลขต่อท้ายดาวด้วยสิ ให้หน้าจอออกมาเป็นแบบนี้ `*1*2*3*4*5` ทำได้ไหมน้า?" (ใบ้: ก็แค่ใช้ความรู้จากข้อสูตรคูณ ให้พิมพ์ตัวแปร `i` ปนเข้าไปด้วย!)
