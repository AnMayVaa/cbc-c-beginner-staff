"พี่อยากให้ทุกคนลองนึกภาพตามนะครับ สมมติว่าพี่เป็นคุณครูที่ต้องเก็บคะแนนสอบของนักเรียน 1 คน พี่ก็แค่สร้างตัวแปร `int score1;` ง่ายๆ ใช่ไหมครับ? แต่ถ้าห้องเรียนของพี่มีนักเรียน 50 คนล่ะ? พี่จะต้องสร้าง `int score1;`, `int score2;`, `int score3;` ไปจนถึง `int score50;` เลยไหม? โอ้โห แค่คิดก็เมื่อยมือแล้วครับ กว่าจะพิมพ์เสร็จคงหมดเวลาเรียนพอดี!"

"นี่คือปัญหาที่ทำให้เกิด 'อาร์เรย์' ขึ้นมาครับ! อาร์เรย์ในโลกการเขียนโปรแกรม ก็เหมือนสิ่งของหลายอย่างใน 'ชีวิตจริง' ที่ถูกจัดเรียงเป็นหมวดหมู่หรือเป็นแถวเดียวกันครับ เช่น ลองนึกถึง 'ตู้ล็อกเกอร์ที่สวนน้ำ' ที่เรียงติดกันเป็นแถวยาวๆ หรือ 'ตู้จดหมายในคอนโด' ที่มีช่องเรียงกันเป็นระเบียบ หรือแม้แต่ 'ที่นั่งในโรงหนัง' ที่เรียงกันเป็นแถวๆ"

"อาร์เรย์พวกนี้สามารถมีหลายมิติด้วยนะครับ! ยกตัวอย่างเช่น ถ้าเรามีแค่ 'ตู้ล็อกเกอร์แถวเดียว' เราจะเรียกว่า อาร์เรย์ 1 มิติ (1D Array) แต่ในชีวิตจริง เรามักเจอ 'อาร์เรย์หลายมิติ (Multi-dimensional Arrays)' ด้วย เช่น แผ่นงาน Excel ที่มีทั้งแถวและคอลัมน์ (ตารางแบบ 2 มิติ), กระดานหมากรุก, หรือแม้แต่หมายเลขห้องในคอนโด ที่มีทั้ง 'ชั้นที่เท่าไหร่' และ 'ห้องที่เท่าไหร่' ประกอบกัน"

"แต่ไม่ต้องตกใจไปครับ! สำหรับค่ายนี้ เพื่อให้พวกเราเข้าใจพื้นฐานได้ง่ายที่สุด เราจะโฟกัสและใช้งานแค่ **อาร์เรย์ 1 มิติ (1D Array)** หรือตู้ล็อกเกอร์แถวเดียวกันไปก่อนนะครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อใหญ่:** อาร์เรย์ (Arrays) — ในชีวิตจริงและหลายมิติ
- **รูปภาพเปรียบเทียบ (Visual Analogy):**
  - ภาพที่ 1: ตู้ล็อกเกอร์เรียงต่อกันเป็นแถว (อาร์เรย์ 1 มิติ)
  - ภาพที่ 2: ตาราง Excel, กระดานหมากรุก หรือ ตู้จดหมายคอนโดหลายชั้น (อาร์เรย์หลายมิติ)
- **ปัญหาที่นำไปสู่อาร์เรย์:** แสดงโค้ด `int score1; int score2; ... int score50;` แล้วมีเครื่องหมายกากบาทสีแดงตัวใหญ่ทับ
- **ไฮไลต์ (Text):** แคมป์นี้เราจะใช้แค่ **อาร์เรย์ 1 มิติ (1D Array)** เท่านั้น!

**💡 เทคนิคการสอน:**
- **การใช้ภาษาท่าทาง (Body Language):** ชี้ให้เด็กเห็นภาพความแตกต่างระหว่าง 1 มิติ (ทำมือวาดแนวนอนเส้นเดียว) กับ 2 มิติ (ทำมือวาดเป็นตารางสี่เหลี่ยม)
- **คลายความกังวล:** ย้ำว่า 2D และ 3D Array อาจจะดูซับซ้อน แต่เรายังไม่ต้องไปแตะมัน ให้เด็กสบายใจว่าเราจะเรียนแค่แบบง่ายๆ ก่อน

## 📄 หน้าที่ 53: อาร์เรย์ (Arrays)

**🎤 บทพูด (สิ่งที่ต้องพูด):**

"ก่อนจะไปหัวข้อสุดท้าย เรามาทำความรู้จักกับตัวแปรแบบพิเศษกันก่อนครับ อย่างที่พี่บอกไปตะกี้ว่า ถ้าเรามีนักเรียน 50 คน การสร้างตัวแปรทีละตัวมันเหนื่อยเกินไป!"

"นี่คือที่มาของ **อาร์เรย์ (Array)** ครับ อาร์เรย์คือการ 'จองกล่องล็อกเกอร์เรียงติดกัน' สมมติเราบอกว่า `int scores[50];` คอมพิวเตอร์จะสร้างล็อกเกอร์ 50 ตู้ติดกันให้เราทันที! สิ่งสำคัญที่สุดที่ต้องจำไว้เลยคือ **ตู้แรก เริ่มที่หมายเลข 0 เสมอ!** ไม่ใช่ 1 นะครับ ดังนั้นตู้สุดท้ายคือเบอร์ 49 ครับ ตู้ล็อกเกอร์เบอร์ 0, 1, 2... เราเรียกว่า Index (ดัชนี) ครับ"

"แล้วมันดียังไงล่ะ? มันดีตรงที่เราสามารถเอา `For Loop` มาใช้ร่วมกับอาร์เรย์ได้ครับ! เราให้ตัวแปรลูป `i` วิ่งตั้งแต่ 0 ถึง 49 แล้วสั่ง `scanf("%d", &scores[i]);` แค่โค้ดสั้นๆ เพียงไม่กี่บรรทัด เราก็สามารถเก็บคะแนนเด็ก 50 คนได้หมดเลย! นี่แหละความมหัศจรรย์ของอาร์เรย์ครับ"

"ทีนี้มาพูดถึง **ข้อความ (String)** กันบ้างครับ น้องๆ อาจจะสงสัยว่า อ้าว แล้วถ้าอยากเก็บชื่อคนล่ะ จะทำยังไง?"
"ในภาษา C แตกต่างจากภาษาใหม่ๆ อย่าง Python ตรงที่มัน **ไม่มีตัวแปรชนิด String ตรงๆ** ให้ใช้ครับ! เซอร์ไพรส์ไหมล่ะ!"

## 📄 หน้าที่ 54: อาร์เรย์ (Arrays) และ String (ข้อความ)

"จำ `char` ได้ไหมครับ? `char` เก็บได้แค่ตัวอักษร 1 ตัว เช่น 'A', 'B' ดังนั้นวิธีแก้ปัญหาของภาษา C คือการเอา `char` หลายๆ ตัวมาเรียงติดกันเป็นอาร์เรย์นั่นเองครับ! เช่น คำว่า `CAT` มันก็คืออาร์เรย์ของ `char` ที่ช่อง 0 (Index 0) เก็บตัวอักษร 'C', ช่อง 1 เก็บ 'A', ช่อง 2 เก็บ 'T' เอามาต่อกันเรียงเป็นสาย จึงกลายเป็น String ครับ!"

"เวลาเราอยากสร้างตัวแปร String เพื่อเก็บชื่อ เราเลยเขียนแบบนี้ครับ: `char name[10];` แปลว่าจองกล่องตัวอักษรเรียงติดกัน 10 กล่องเพื่อเตรียมเก็บชื่อนั่นเอง"

"**มีจุดสำคัญมากๆ เวลาใช้ String กับคำสั่ง scanf ครับ ฟังดีๆ นะ:**"
"ปกติเวลาใช้ scanf เราต้องใส่เครื่องหมาย `&` นำหน้าชื่อตัวแปรใช่ไหมครับ? แต่สำหรับ String (ที่สร้างจาก `char[]`) เรา **ไม่ต้องใส่ &** นะครับ! เราพิมพ์แค่ `scanf("%s", name);` ได้เลย! เพราะอะไร? เพราะชื่อของอาร์เรย์มันแอบเก็บที่อยู่ (Address) จุดเริ่มต้นของตู้ล็อกเกอร์ตู้อรแกไว้เรียบร้อยแล้วครับ มันเป็นข้อยกเว้นเพียงหนึ่งเดียวที่น้องๆ ต้องจำให้แม่นๆ นะครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อใหญ่:** อาร์เรย์ (Arrays) และ String
- **ภาพประกอบ Array:** ตู้ล็อกเกอร์เรียงติดกัน มีหมายเลข(Index) แปะหน้าตู้เริ่มจาก 0, 1, 2, 3...
- **ตัวอย่างโค้ด Array ทั่วไป:** `int scores[5] = {10, 20, 30, 40, 50};` พร้อมภาพล็อกเกอร์ที่มีตัวเลขอยู่ข้างใน
- **String = Array of char:** อธิบายการนำ `char` มาเรียงติดกัน เช่น 'C', 'A', 'T' ในกล่อง index 0, 1, 2
- **ตัวอย่างโค้ด String:**
```c
char name[20];
printf("Enter your name: ");
scanf("%s", name); // ⚠️ ไม่มี &
printf("Hello %s!\n", name);
```
- **กล่องเตือนสีแดง (จุดเน้นย้ำ):**
  1. Index ของ Array เริ่มที่ 0 เสมอ!
  2. `scanf` กับ `%s` (String) **ไม่ต้องใส่ `&`**

**💡 เทคนิคการสอน:**
- **เล่ามุก:** โปรแกรมเมอร์เริ่มนับเลขที่ 0 เสมอ! ให้เด็กลองนับนิ้วตัวเองดู 0,1,2,3,4 ถ้าใครนับ 1 ก่อนแสดงว่ายังไม่ใช่โปรแกรมเมอร์ที่แท้จริง! (เรียกเสียงหัวเราะ)
- **ทวนซ้ำดังๆ หน้าคลาส:** "รับตัวเลขใช้ & ไหม?" (เด็กตอบ: ใช้!) "รับตัวอักษรตัวเดียวใช้ & ไหม?" (เด็กตอบ: ใช้!) "รับ String ใช้ & ไหม?" (เด็กตอบ: ไม่ใช้!) การให้เด็กตะโกนตอบพร้อมกันจะช่วยให้จำแม่นขึ้น

## 📄 หน้าที่ 55: 🏋️ แบบฝึกหัด 4.1: เครื่องคิดเงินรวม (Cash Register)

**📝 โจทย์:** รับราคาสินค้าจำนวน 5 ชิ้น เก็บข้อมูลลงใน Array จากนั้นใช้ For Loop บวกรวมราคาทั้งหมด แล้วแสดงยอดสุทธิ

**🎤 บทพูด (สิ่งที่ต้องพูด):**

"มาถึงบททดสอบความเข้าใจเรื่องอาร์เรย์และลูปแบบแพ็คคู่กันแล้วครับ! ข้อนี้เป็นรูปแบบการเขียนโค้ดที่โปรแกรมเมอร์เจอบ่อยมากๆ เราเรียกแพทเทิร์นนี้ว่า **Accumulator Pattern** หรือ 'รูปแบบการสะสมยอด' นั่นเอง"

"ลองจินตนาการแบบนี้ครับ น้องๆ เป็นพนักงานแคชเชียร์ยืนอยู่หน้าเครื่องคิดเงินที่ 7-Eleven มีลูกค้าหอบขนมมาวางตู้ม 5 ชิ้น น้องต้องหยิบเครื่องสแกนมา 'ติ๊ด... ติ๊ด...' ทีละชิ้นๆ แล้วราคาแต่ละชิ้นมันก็จะถูกบวกสะสมรวมกันไว้ในเครื่องคิดเงินใช่ไหมครับ? พอติ๊ดครบ 5 ชิ้น เครื่องก็จะสรุปยอดรวม (Total) ออกมา"

"ลอจิกการทำงานของโปรแกรมเราก็จำลองมาจากเครื่องคิดเงินเป๊ะๆ เลยครับ:"
"1. สร้าง Array ชื่อ `prices` ขนาด 5 ช่อง เพื่อเปรียบเสมือนตะกร้าเก็บราคาสินค้าทั้ง 5 ชิ้น"
"2. สร้างตัวแปรลูปที่ 1 — ทำหน้าที่เหมือนตอนพนักงานสแกนบาร์โค้ด วนรับค่า 5 รอบ เก็บเข้า Array ไปทีละช่อง"
"3. สร้างตัวแปรชื่อ `sum` เพื่อใช้เป็น 'ยอดรวมในเครื่องคิดเงิน' โดย **ต้องกำหนดค่าเริ่มต้นให้เป็น 0 เสมอ!** สำคัญมากนะ!"
"4. สร้างตัวแปรลูปที่ 2 — ให้มันวิ่งกวาดเอาของในตะกร้า Array ทีละช่องๆ มา 'บวกทบสะสม' เข้าไปในตัวแปร `sum`"
"5. สิ้นสุดลูป ก็พิมพ์ `sum` ออกมาเป็นยอดสุทธิ"

"**คำถามสำคัญ: ทำไมเราต้องกำหนดให้ `int sum = 0;` ตอนเริ่มต้นด้วยล่ะ? ไม่ใส่ `= 0` ได้ไหม?**"
"คำตอบคือ 'ไม่ได้เด็ดขาดครับ!' เพราะในภาษา C ถ้าเราสร้างตัวแปรมาลอยๆ แบบไม่ได้บอกค่าให้มัน มันอาจจะมี 'ค่าขยะ (Garbage Value)' ซึ่งเป็นเศษข้อมูลเก่าๆ ที่หลงเหลืออยู่ใน RAM ของคอมพิวเตอร์ติดมาด้วย! ลองนึกดูว่าถ้าเปิดเครื่องคิดเงินมา มีค่าขยะค้างอยู่ 12,345 บาท ลูกค้าซื้อลูกอมเม็ดเดียว 5 บาท รวมบิลออกมา 12,350 บาท ลูกค้าโวยวายบ้านแตกแน่นอนครับ! เพราะฉะนั้นเราต้อง 'เคลียร์เครื่องคิดเงินให้เป็น 0 เสมอ' ก่อนเริ่มสะสมยอดครับ!"

"มาดูโค้ดกันเลย:"
```c
#include <stdio.h>

int main() {
    int prices[5]; // ตะกร้า 5 ช่อง
    int sum = 0;   // ⚠️ เครื่องคิดเงิน ต้องเริ่มที่ศูนย์!

    // ลูปที่ 1: รับราคาสินค้า 5 ชิ้น
    printf("--- Enter 5 Prices ---\n");
    for (int i = 0; i < 5; i++) {
        printf("Item %d: ", i + 1);
        scanf("%d", &prices[i]); // เก็บลงตะกร้าช่องที่ i
    }

    // ลูปที่ 2: ดึงของในตะกร้ามาบวกสะสมยอดรวม
    for (int i = 0; i < 5; i++) {
        sum = sum + prices[i];  // สะสมยอดทีละชิ้น
    }

    // พิมพ์ใบเสร็จยอดรวม
    printf("\nTotal: %d baht\n", sum);
    return 0;
}
```

"มาไล่ดูความมหัศจรรย์ของลูปสะสมยอด (Accumulator loop) กันครับ สมมติลูกค้าซื้อของราคา 10, 20, 30, 40, 50 บาทตามลำดับ:"
"- ตอนเริ่มต้นก่อนเข้าลูป: เครื่องคิดเงิน `sum = 0`"
"- รอบ 1 (`i=0`): `sum` ตัวใหม่ = (0 เดิม) + (ของชิ้นแรก 10) = 10"
"- รอบ 2 (`i=1`): `sum` ตัวใหม่ = (10 เดิม) + (ของชิ้นที่สอง 20) = 30"
"- รอบ 3 (`i=2`): `sum` ตัวใหม่ = (30 เดิม) + (ของชิ้นที่สาม 30) = 60"
"- รอบ 4 (`i=3`): `sum` ตัวใหม่ = (60 เดิม) + (ของชิ้นที่สี่ 40) = 100"
"- รอบ 5 (`i=4`): `sum` ตัวใหม่ = (100 เดิม) + (ของชิ้นที่ห้า 50) = 150"
"- จบลูป! เครื่องคิดเงินสรุปยอดออกมาเป๊ะๆ คือ 150 บาทครับ!"

"อีกนิดนึงครับ สังเกตตอนรับค่าด้วย scanf ไหมครับ? — `scanf("%d", &prices[i]);` ตรง `prices[i]` คือการเจาะจงชี้ไปที่ Array ช่องหมายเลข `i` ดังนั้น การใส่ `&prices[i]` ก็คือการส่ง 'ที่อยู่ของช่องนั้นๆ' ให้ scanf เอาของไปหย่อนให้ถูกที่นั่นเองครับ"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **Pattern การทำงาน (Algorithm):** สร้าง Array → ลูปรับค่า → สร้างตัวแปร sum=0 → ลูปสะสมยอด → แสดงผล
- **กล่องเตือนความจำ (ย้ำบ่อยๆ):** ⚠️ ก่อนจะใช้ตัวแปรแบบ Accumulator (สะสม) **ต้องตั้งค่า `sum = 0` เสมอ!** ห้ามลืมเด็ดขาด ป้องกันปัญหา Garbage Value!
- **สมการสะสมยอด:** `sum = sum + prices[i];` (ยอดใหม่ = ยอดเดิม + ของชิ้นปัจจุบัน)
- **Trace Table การสะสมยอด:** โชว์ตารางแสดงการพอกพูนของค่า sum ในแต่ละรอบลูป (0→10→30→60→100→150)
- **โค้ดเต็ม + Output ตัวอย่าง**

**💡 เทคนิคการสอน:**
- **แสดงให้เห็นหายนะของ Garbage Value:** ให้เด็กดื้อแพ่งลองลบ `= 0` ออกจากตัวแปร sum แล้วรันโปรแกรมดูหลายๆ รอบ ให้เด็กเห็นว่าตัวเลขประหลาดโผล่มาจากไหนไม่รู้ เป็นการตอกย้ำให้จำแม่นๆ
- **ใช้ Visual Aid ทรงพลัง:** วาดตาราง Trace Table ให้เห็นการสะสมค่าของ sum ขึ้นกระดาน มันจะอิมแพคมาก
- **อวดความโปรนิดนึง:** อธิบายให้เด็กเก่งฟังว่า บรรทัด `sum = sum + prices[i];` โปรแกรมเมอร์สายย่อ มักจะเขียนสั้นๆ ว่า `sum += prices[i];` ความหมายเหมือนกันเป๊ะ! ใครอยากเขียนแบบโปรๆ ลองใช้แบบย่อได้เลย

## 📄 หน้าที่ 56: 🏋️ แบบฝึกหัด 4.2: หาค่าเฉลี่ย (Find Average)

**📝 โจทย์:** ต่อยอดจากข้อ 4.1 — เมื่อได้ผลรวม (sum) แล้ว ให้นำมาหาค่าเฉลี่ยราคาสินค้าต่อชิ้น (Average) และแสดงผลทศนิยม 2 ตำแหน่ง

**🎤 บทพูด (สิ่งที่ต้องพูด):**

"ข้อนี้เราจะมาต่อยอดจากแคชเชียร์ข้อที่แล้วครับ เรามี `sum` หรือยอดรวมอยู่แล้วเรียบร้อย สูตรหาค่าเฉลี่ยทางคณิตศาสตร์ก็คือเอา 'ผลรวม หารด้วย จำนวนของ' ซึ่งก็คือ `sum / 5` นั่นเอง ฟังดูง้ายง่าย... กล้วยๆ... แต่ช้าก่อน! **มันมีกับดักคำโตซ่อนอยู่ครับ!**"

"กับดักนี้ทำให้เด็กคอมหลายคนสอบตกมานักต่อนักแล้วครับ ฟังดีๆ นะ... ถ้าเรามียอดรวม `sum` เป็นชนิด int แล้วเราหารด้วยเลข 5 ซึ่งก็เป็นชนิด int ธรรมชาติของคณิตศาสตร์เช่น สมมติ `sum = 150` เอา `150 / 5` มันได้ 30 ถ้วนพอดี ไม่มีปัญหาอะไรใช่ไหมครับ?"
"แต่... แล้วถ้าของมันราคา 153 บาทล่ะ? ในโลกความเป็นจริง `153 / 5` มันต้องได้ `30.6` ถูกไหม? แต่ในโลกของภาษา C ถ้าเราเอาประเภทจำนวนเต็ม (int) ไปหาร จำนวนเต็ม (int) มันจะบอกว่า 'งั้นฉันก็จะคืนคำตอบเป็นจำนวนเต็มให้ละกัน' **มันจะตัดเศษทศนิยมทิ้งดื้อๆ เลยครับ!** คำตอบที่ได้จะออกมาเป็น 30 ถ้วนๆ หายไปเลย 0.6 บาท!"

"ถ้าเราเขียนแค่ `float average = sum / 5;` เราจะไม่มีวันได้เศษทศนิยมเลยครับ!"

"วิธีแก้คำสาปนี้คือ — เราต้องหลอกคอมพิวเตอร์ หรือ 'บังคับ' ให้มันแปลงตัวแปรข้างใดข้างหนึ่งของการหารให้กลายเป็น `float` (ทศนิยม) เสียก่อน! พอมีทศนิยมโผล่มาในสมการปุ๊บ คอมพิวเตอร์ถึงจะยอมหารออกมาแบบละเอียดให้ครับ"

"เราทำได้ 2 วิธีง่ายๆ ครับ:"
"**วิธีที่ 1 (สายอ้อมๆ):** แทนที่เราจะหารด้วย `5` เฉยๆ เราเปลี่ยนไปหารด้วย `5.0` แทนครับ!"
"`float average = sum / 5.0;`"
"เพราะเลข `5.0` มีจุดทศนิยม คอมพิวเตอร์ถือว่ามันเป็นข้อมูลชนิด float ทันที กฎของภาษา C มีอยู่ว่า `int หาร float จะได้ผลลัพธ์ละเอียดเป็น float เสมอ` ครับ"

"**วิธีที่ 2 (สายโปรแกรมเมอร์ตัวจริง):** เราใช้เวทมนตร์ที่เรียกว่า **Type Casting (การแปลงร่างชนิดข้อมูล)** ครับ!"
"`float average = (float)sum / 5;`"
"การที่เราเอา `(float)` ไปแปะไว้หน้าตัวแปร `sum` เป็นการกระซิบสั่งคอมพิวเตอร์ว่า 'นี่ๆๆ เปลี่ยนร่าง sum จากจำนวนเต็มให้กลายเป็นทศนิยมชั่วคราวนะ แล้วค่อยเอาไปหาร 5'"

"เรามาดูโค้ดที่เติมส่วนการหาค่าเฉลี่ยเข้าไปกันครับ:"
```c
#include <stdio.h>

int main() {
    int prices[5];
    int sum = 0;

    // ... (ส่วนรับค่า และบวกสะสมค่า เหมือนข้อ 4.1 ทุกประการ) ...

    // ดึงค่าผลรวมมาคำนวณหาค่าเฉลี่ย (อย่าลืม Type Casting ระวังกับดัก int/int)
    float average = (float)sum / 5;  // หรือจะใช้ sum / 5.0 ก็ได้ผลเหมือนกัน
    
    printf("Total: %d baht\n", sum);
    printf("Average: %.2f baht\n", average); // แสดงผลเป็นทศนิยม 2 ตำแหน่ง
    return 0;
}
```

"จำไว้นะครับ เป็นกฎเหล็กเลย: **int หาร int เท่ากับ int เสมอ (เศษทิ้งหมด!)** จะหารเลขหาค่าเฉลี่ยเมื่อไหร่ นึกถึงกับดักข้อนี้ทุกครั้งนะครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **Warning Label (กากบาทตัวเบ้อเริ่ม):** กับดักมรณะ `int / int = int` (ตัดเศษทิ้งไม่เหลือเยื่อใย!) ตัวอย่าง: `153/5 = 30` (ผิด!) ไม่ใช่ `30.6`!
- **Solution (วิธีรอดชีวิต):**
  - ทางเลือก 1: เปลี่ยนตัวหารเป็นทศนิยม `sum / 5.0`
  - ทางเลือก 2: ใช้เวทมนตร์ Type Casting `(float)sum / 5`
- **โค้ดเต็ม + Output ทดสอบ:** รันด้วยยอดเงินรวมที่ไม่ลงตัว (เช่น 153 บาท) ให้เห็นว่าค่าเฉลี่ยออกมาเป็นทศนิยมจริงๆ (30.60 baht)

**💡 เทคนิคการสอน:**
- **ดักทางเด็ก:** ให้เด็กลองเขียนแบบไร้เดียงสาไปก่อน `float avg = sum / 5;` แล้วสั่งให้เด็กกรอกเลขที่บวกกันแล้วไม่หาร 5 ไม่ลงตัว (เช่น ยอด 12) พอผลออกมาเป็น 2.00 (แทนที่จะเป็น 2.40) เด็กจะตกใจและเกิดความสงสัย แล้วเราค่อยทำตัวเป็นฮีโร่เข้ามาชี้ทางสว่างให้
- **เช็คความเข้าใจด้วยเลขง่ายๆ:** ตั้งคำถามว่า "น้องๆ ครับ ในภาษา C ถ้าพี่พิมพ์สมการนี้ `7 / 2` พี่จะได้ผลลัพธ์เป็นเท่าไหร่ครับ?" (รอเด็กงงๆ ตอบ 3.5 แล้วรีบตอกกลับว่า "ผิด! ได้ 3 ถ้วนๆ ครับ! อย่าลืมกับดักเราสิ!")

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อใหญ่:** ฟังก์ชัน (Functions) — User-Defined Functions
- **รูปภาพเปรียบเทียบ (Visual Analogy):**
  - ภาพโค้ดยาวๆ ที่พันกันยุ่งเหยิง (สปาเก็ตตี้โค้ด) พร้อมกากบาทสีแดง ❌
  - ภาพเครื่องจักรหรือโรงงานเล็กๆ ที่มีท่อส่งของเข้า (Input) ตรงกลางมีฟันเฟืองหมุนๆ (Process) และท่อส่งของออก (Output) พร้อมเครื่องหมายถูกสีเขียว ✅
- **Keyword สำคัญ:** Reusability (นำกลับมาใช้ใหม่ได้โดยไม่ต้องเขียนซ้ำ), Modular (แบ่งเป็นส่วนๆ ให้อ่านง่าย)

- **สร้างความตื่นเต้น:** บอกเด็กๆ ว่า "นี่คือหัวข้อที่เป็นเส้นแบ่งระหว่างนักเรียนที่แค่เขียนโค้ดได้ กับ โปรแกรมเมอร์มืออาชีพ เลยนะ! ใครทำฟังก์ชันเป็นถือว่าก้าวไปอีกขั้นแล้ว!"

## 📄 หน้าที่ 62: ฟังก์ชัน (FUNCTION) — แบบที่ 1: ฟังก์ชันที่ไม่คืนค่า (ชนิด `void`)

"**แบบที่ 1: ฟังก์ชันที่ไม่คืนค่า (ชนิด `void`)**"
"ใช้เมื่อเราแค่อยากให้ฟังก์ชันไปทำอะไรสักอย่างให้เสร็จ โดยไม่ต้องส่งผลลัพธ์อะไรกลับมาให้เรา เช่น ให้มันไปพิมพ์ข้อความต้อนรับ"

```c
void say_hello() {
    printf("Hello CBC!\n");
}
```

"คำว่า `void` ในภาษาอังกฤษแปลว่า 'ว่างเปล่า' ครับ แปลว่าฟังก์ชันนี้ไม่มีอะไร return กลับออกมา เหมือนเราสั่งให้เพื่อนไปปิดไฟ เพื่อนเดินไปปิดไฟให้เสร็จแล้วก็จบ ไม่ได้เอาอะไรติดมือกลับมาให้เรา"

## 📄 หน้าที่ 63: ฟังก์ชัน (FUNCTION) — แบบที่ 1: ฟังก์ชันที่ไม่คืนค่า (ชนิด `void`)

"**แบบที่ 2: ฟังก์ชันที่คืนค่า (ต้องมี `return`)**"
"ใช้เมื่อเราอยากให้ฟังก์ชันช่วยคำนวณอะไรสักอย่าง แล้วส่งผลลัพธ์กลับมาให้เราเอาไปใช้งานต่อ"
```c
int add(int a, int b) {
    int result = a + b;
    return result;
}
```
"ตัวนี้ประกาศเป็นชนิด `int` แปลว่าต้องส่งตัวเลขจำนวนเต็มกลับมา มันรับ `a` กับ `b` เข้าไป (นี่คือ Parameters/ผลไม้) แล้วเอามาบวกกัน (นี่คือ Process/เครื่องปั่น) แล้ว `return` ผลลัพธ์กลับมา (นี่คือ Output/น้ำผลไม้)"

"คราวนี้มาดูโค้ดเต็มๆ เวลาเราเอาฟังก์ชันไปใช้งานร่วมกับ `main()` กันครับ:"
```c
#include <stdio.h>

// ประกาศฟังก์ชันไว้ข้างบน main เสมอ!
void say_hello() {
    printf("Hello CBC!\n");
}

int add(int a, int b) {
    int result = a + b;
    return result;
}

int main() {
    say_hello();  // เรียกใช้ say_hello เฉยๆ ไม่ต้องมีตัวแปรมารับ

    int sum = add(5, 7);  // เรียกใช้ add โยน 5 กับ 7 เข้าไป แล้วเอาตัวแปร sum มารับผลลัพธ์
    printf("5 + 7 = %d\n", sum);

    return 0;
}
```

"สิ่งที่น้องๆ ต้องจำให้ขึ้นใจเลยคือ:"
"1. **ฟังก์ชันต้องประกาศไว้ 'ข้างบน' main เสมอ** เพราะคอมพิวเตอร์อ่านโค้ดจากบรรทัดบนลงล่างครับ ถ้าเราไปเรียกใช้ฟังก์ชันก่อนที่คอมพิวเตอร์จะรู้จักมัน คอมพิวเตอร์จะงงแล้วก็ Error ทันที!"
"2. **Parameters คือของที่ส่งเข้าไป** — ต้องบอกทั้งชนิดข้อมูลและชื่อตัวแปร เช่น `int a, int b`"
"3. **return คือของที่ส่งกลับออกมา** — ชนิดข้อมูลที่ return ต้องตรงกับชนิดที่ประกาศไว้ที่ชื่อฟังก์ชัน (ฟังก์ชันชื่อชนิด int ก็ต้อง return ค่า int)"
"4. **void = ไม่ return อะไร** ฟังก์ชันทำงานเสร็จแล้วก็จบกันไป"

"มีน้องบางคนชอบถามว่า — 'พี่ครับ ทำไมต้องวุ่นวายแยกเป็นฟังก์ชันด้วย? เขียนรวดเดียวใน main ก็ได้นี่นา?'"
"คำตอบคือ — ลองจินตนาการว่าเราต้องเขียนโค้ดสุ่มตัวเลข 20 ครั้ง ใน 20 ที่ที่แตกต่างกัน ถ้าไม่มีฟังก์ชัน เราต้องก็อปปี้โค้ดสุ่มเลขยาวๆ วางซ้ำ 20 ครั้ง! โค้ดจะยาวเหยียด แต่ถ้าเรามีฟังก์ชัน เราแค่เรียกชื่อมัน 20 ครั้ง โค้ดสั้นลงเยอะมาก! และถ้าวันหนึ่งเราอยากเปลี่ยนวิธีสุ่มเลข เราแค่ไปแก้โค้ดที่ตัวฟังก์ชันที่เดียวจบเลย ไม่ต้องตามไปแก้ทั้ง 20 จุดครับ! นี่แหละความสุดยอดของฟังก์ชัน!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อใหญ่:** ฟังก์ชัน (Functions) — เจาะลึกการสร้างเครื่องปั่น
- **ภาพเปรียบเทียบ:** เครื่องปั่นน้ำผลไม้ (มีลูกศรชี้ให้เห็น Input → Black Box(Process) → Output)
- **ตารางเปรียบเทียบ 2 แบบ:**

| แบบฟังก์ชัน | คำอธิบาย | ตัวอย่างการสร้าง | เมื่อไหร่ถึงใช้ |
|---|---|---|---|
| `void` | ไม่คืนค่า (ว่างเปล่า) | `void say_hello()` | ต้องการแค่ให้ทำงานบางอย่างให้จบ ไม่ต้องส่งค่าอะไรกลับ |
| `int`/`float` | มีการคืนค่า (`return`) | `int add(int a, int b)` | ต้องการให้คำนวณและส่งผลลัพธ์กลับมาใช้งานต่อ |

- **ตัวอย่างโค้ดเต็ม:** แสดงโครงสร้างโปรแกรมที่มีทั้งฟังก์ชัน void, ฟังก์ชัน int และ `main()` อย่างชัดเจน
- **กฎเหล็ก (ตัวหนาสีแดง):** ฟังก์ชันต้องประกาศไว้ "ข้างบน" `main()`!

**💡 เทคนิคการสอน:**
- **วาดรูปประกอบบนกระดาน:** วาดรูปกล่องสี่เหลี่ยม (Black Box) มีลูกศรชี้เข้า (Input/Parameters) และลูกศรชี้ออก (Output/Return) ให้เด็กคุ้นชินกับ Flow การทำงาน
- **ฝึกการคิด:** ลองตั้งโจทย์สดๆ เช่น "ถ้าพี่อยากสร้างฟังก์ชันคำนวณพื้นที่สี่เหลี่ยมผืนผ้า (กว้าง x ยาว) น้องๆ คิดว่า Input คืออะไร? แล้ว Return ควรเป็นข้อมูลชนิดไหน?" (Input: กว้าง, ยาว / Return: int หรือ float)
- **จำลอง Error:** ชี้ให้เด็กดูว่า "ถ้าพี่ดื้อ เอาฟังก์ชัน add ไปเขียนไว้ข้างล่าง main จะเกิดอะไรขึ้น?" (ให้เด็กตอบว่า Error เพราะคอมอ่านจากบนลงล่าง)

## 📄 หน้าที่ 63.5: ฟังก์ชัน (FUNCTION) — การเรียกใช้งาน (Calling a Function)

**🎤 บทพูด (สิ่งที่ต้องพูด):**

"ทีนี้ เมื่อเราสร้างโรงงานหรือเครื่องจักร (ฟังก์ชัน) ของเราเสร็จแล้ว เราจะเอามันมาใช้งานได้ยังไง? การเอามาใช้งาน เราเรียกว่า 'การเรียกใช้ฟังก์ชัน (Calling a Function)' ครับ!"

"เวลาที่เราอยากให้เพื่อนไปซื้อน้ำให้ เราก็แค่เรียกชื่อเพื่อน แล้วให้เงินไปใช่ไหมครับ? ฟังก์ชันก็เหมือนกันครับ เวลาเราจะเรียกใช้ เราแค่พิมพ์ **ชื่อฟังก์ชัน** ตามด้วย **วงเล็บ ( )** ครับ ถ้าฟังก์ชันนั้นต้องการข้อมูล (Parameters) เราก็ใส่ข้อมูลลงไปในวงเล็บให้มันด้วย เช่น โยนเงินให้เพื่อนไปซื้อน้ำนั่นแหละครับ"

"ลองดูตัวอย่างกันครับ ถ้าพี่อยากเรียกใช้เครื่องจักรที่ชื่อ `say_hello` ซึ่งไม่ต้องการ Input อะไรเลย พี่ก็แค่พิมพ์ว่า `say_hello();` แค่นี้เลยครับ สั้นๆ ง่ายๆ!"

"แต่ถ้าพี่จะเรียกเครื่องบวกเลข `add` ที่ต้องการ Input 2 ตัว พี่ก็ต้องส่งตัวเลขให้มันด้วยการพิมพ์ว่า `add(5, 7);` ครับ แล้วอย่าลืมนะครับ! ถ้าฟังก์ชันมันมีการ `return` ผลลัพธ์กลับมา (เหมือนเครื่องปั่นที่คายน้ำผลไม้ออกมา) เราต้องเตรียม **ตัวแปร (แก้วน้ำ)** มารองรับมันด้วยครับ! พี่เลยต้องเขียนเป็น `int sum = add(5, 7);` เพื่อเอาแก้วที่ชื่อ `sum` มารองรับผลบวกนั่นเองครับ!"

"ดังนั้น จำกฎเหล็กของการเรียกใช้งานให้ดีนะครับ: เรียกชื่อมัน, ให้อาหารมัน (ถ้ามันต้องการ), และเตรียมกล่องมารับของจากมัน (ถ้ามันมีการคืนค่า) ครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **หัวข้อใหญ่:** การเรียกใช้งานฟังก์ชัน (Calling a Function)
- **รูปแบบโครงสร้าง (Syntax):** 
  - `ชื่อฟังก์ชัน(ข้อมูลที่ต้องส่ง);`
  - `ตัวแปรรับค่า = ชื่อฟังก์ชัน(ข้อมูลที่ต้องส่ง);`
- **ตัวอย่างโค้ดไฮไลต์:**
  ```c
  // แบบไม่มี Parameter และไม่ Return
  say_hello(); 

  // แบบมี Parameter และมี Return
  int sum = add(5, 7);
  ```
- **รูปภาพเปรียบเทียบ:** รูปคนเรียกเพื่อน "แกๆ ไปซื้อน้ำให้หน่อย (ยื่นเงิน 20 บาท)" → รับน้ำกลับมาใส่ในมือ (ตัวแปร)

**💡 เทคนิคการสอน:**
- **แอคติ้ง:** ให้ผู้สอนเล่นบทบาทสมมติ โดยชี้ไปที่น้องค่ายคนนึงแล้วบอกว่า "สมมติว่าน้องชื่อฟังก์ชัน A พี่บอกว่า 'A ลุกขึ้นยืน' (นี่คือเรียกฟังก์ชันแบบ void ไม่มีตัวแปรรับ) แต่ถ้าพี่บอก 'A เอาปากกามาให้พี่หน่อย' พี่ต้องแบมือรอรับปากกาจาก A ด้วย (นี่คือเรียกฟังก์ชันแบบมี return ต้องมีตัวแปรรอรับ)" เพื่อให้เด็กเห็นภาพการ "เรียกใช้" และ "รับค่า" ชัดเจนขึ้น

## 📄 หน้าที่ 64: 🏋️ แบบฝึกหัด 4.3: ฟังก์ชันทวีคูณ (Double It!)

**📝 โจทย์:** สร้างฟังก์ชันส่วนตัวขึ้นมา 1 ตัวชื่อ `double_number` โดยให้ฟังก์ชันนี้รับตัวเลขจำนวนเต็มมา 1 ตัว แล้วคูณ 2 (ทำเป็นสองเท่า) แล้วส่งผลลัพธ์ (return) กลับคืนมายัง `main()`

**🎤 บทพูด (สิ่งที่ต้องพูด):**

"หลังจากเราเรียนทฤษฎีเรื่องฟังก์ชันมาแล้ว คราวนี้เรามาลอง 'สร้างโรงงาน' เล็กๆ ของตัวเองดูบ้างครับ! โจทย์บอกว่าให้เราสร้างโรงงานที่ชื่อว่า `double_number` หรือแปลว่า 'เครื่องทำสองเท่า' นั่นเอง"

"โรงงานนี้ทำหน้าที่โคตรจะตรงไปตรงมาครับ คือรับตัวเลขเข้ามา (Input) จัดการเอาไปคูณ 2 (Process) แล้วก็ปล่อยตัวเลขที่คูณเสร็จแล้วกลับออกไป (Output/Return)"

"มาแยกชิ้นส่วนเครื่องจักรเครื่องนี้ดูก่อนสร้างจริงครับ:"
"1. **ชื่อฟังก์ชัน:** เราจะตั้งชื่อให้สื่อความหมายว่า `double_number`"
"2. **Parameter (Input ที่รับเข้ามา):** เครื่องนี้ต้องการตัวเลขจำนวนเต็ม 1 ตัว เราจะเปิดช่องรับเป็น `int x`"
"3. **Return Type (ชนิดของผลลัพธ์ที่ส่งกลับ):** เนื่องจากรับเป็น int คูณ 2 เสร็จก็ยังเป็น int อยู่ ดังนั้นเราจะแปะป้ายหน้าเครื่องจักรว่านี่คือฟังก์ชันประเภท `int`"
"4. **กระบวนการทำงานข้างใน (Process):** ง่ายสุดๆ แค่เอา `x * 2`"

"ถ้าเปรียบเป็นเครื่องปั่นน้ำผลไม้อีกครั้ง:"
"- โยนแอปเปิ้ล 1 ลูกเข้าไป (อันนี้คือ `x`)"
"- เครื่องทำงานปั่นๆ แล้วแยกร่างมันเป็น 2 ลูก (`x * 2`)"
"- เสร็จปุ๊บก็คายแอปเปิ้ล 2 ลูกนี้ออกมาทางท่อออก (`return`)"

"มาดูประกอบร่างเป็นโค้ดกันครับ:"
```c
#include <stdio.h>

// 1. สร้างโรงงาน (ฟังก์ชัน) ของเราไว้ 'ข้างบน' main
// บอกว่าเครื่องนี้คืนค่า int, ชื่อ double_number, รับวัตถุดิบเป็น int x
int double_number(int x) {
    int result = x * 2;  // ทำเป็น 2 เท่า
    return result;       // ส่งผลลัพธ์คืนออกไปนอกโรงงาน
    
    // (จริงๆ เขียนย่อบรรทัดเดียวเป็น return x * 2; เลยก็ได้นะ!)
}

int main() {
    int num;
    printf("Enter a number: ");
    scanf("%d", &num);

    // 2. เรียกใช้งานโรงงาน! 
    // โยน num เข้าไป แล้วเอาตัวแปร answer มารับผลลัพธ์ที่ถูกพ่นออกมา
    int answer = double_number(num);
    
    printf("%d doubled is %d\n", num, answer);
    return 0;
}
```

"ลองจำลองการทำงานดูนะ สมมติผู้ใช้ป้อนเลข `7`"
"- โปรแกรมใน `main` จะร้องเรียกฟังก์ชัน `double_number(7)`"
"- การทำงานจะกระโดดเด้งดึ๋งไปที่ฟังก์ชันข้างบน เอาเลข 7 ไปหย่อนใส่ตัวแปร `x`"
"- โรงงานข้างบนจะคำนวณ `7 * 2 = 14`"
"- เสร็จแล้วมันจะ `return 14` (พ่นเลข 14) กลับลงมาที่ `main`"
"- ตัวแปร `answer` ที่อ้าแขนรอรับอยู่ข้างล่าง ก็จะได้เลข 14 ไปครอบครอง"
"- สุดท้ายก็พิมพ์ออกมาว่า '7 doubled is 14'"

"เห็นไหมครับว่า ฟังก์ชันมันเรียบง่ายมากๆ แค่ 3 ส่วนหลัก: ประกาศสเปค, คำนวณ, และ return แค่นั้นเอง! ง่ายกว่าที่คิดเยอะเลยใช่ไหม?"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **ชำแหละส่วนประกอบฟังก์ชัน:** ชื่อ (`double_number`), Parameter (`int x`), Return type (`int`), และ Process (`x * 2`)
- **โค้ดเต็ม + Output ตัวอย่าง** (กรอก 7 ได้ 14)
- **กราฟิก Diagram การเรียกใช้งาน (Flow of Execution):** แสดงเส้นลูกศรวิ่งจาก `main()` → โยนเลข 7 ขึ้นไปที่ฟังก์ชัน `double_number` → แล้วมีลูกศรเด้งกลับพ่นเลข 14 กลับมาที่ `main()` ภาพนี้จะช่วยคลายความงงให้เด็กได้อย่างยอดเยี่ยม

**💡 เทคนิคการสอน:**
- **ท้าทายเด็กสายสร้างสรรค์:** เมื่อทำเสร็จ ให้มอบมิชชั่นใหม่ "เยี่ยมมาก! ไหนใครลองสร้างฟังก์ชันเครื่องจักรตัวที่ 2 ชื่อว่า `triple_number` ที่คูณ 3 ให้พี่ดูหน่อยสิ! แล้วเรียกใช้มันใน main ด้วยเลยนะ"

## 📄 หน้าที่ 65: 🏋️ แบบฝึกหัด 4.4: ฟังก์ชันตรวจสอบผ่าน/ตก (check_pass)

**📝 โจทย์:** สร้างฟังก์ชันที่ล้ำขึ้นมาอีกขั้น ชื่อ `check_pass` รับคะแนนมาพิจารณา ถ้าคะแนน >= 50 ให้ฟังก์ชัน return เลข 1 (มีความหมายแฝงว่า 'สอบผ่าน') แต่ถ้าคะแนน < 50 ให้ return เลข 0 (แฝงความหมายว่า 'สอบตก')

**🎤 บทพูด (สิ่งที่ต้องพูด):**

"เดินทางมาถึงข้อสุดท้ายของบทเรียนนี้แล้วครับ! ข้อนี้พิเศษและ Advance ขึ้นมาอีกขั้น เพราะเรากำลังจะ 'นำ 2 สิ่งที่ทรงพลังที่สุด' มาหลอมรวมกัน นั่นคือ 'การสร้างฟังก์ชัน' + 'การใช้เงื่อนไข if-else' เอามาใส่ไว้ข้างในฟังก์ชันครับ!"

"ลอจิกที่เราต้องการให้ฟังก์ชันตัวนี้ทำคือ เป็นเหมือนครูตรวจข้อสอบครับ รับคะแนนของนักเรียนเข้ามา 1 ก้อน แล้วทำการตัดสินใจด้วยตัวเอง:"
"- ถ้าคะแนน (score) มากกว่าหรือเท่ากับ 50 → ให้ส่งเลข 1 กลับไป (เป็นตัวแทนของคำว่า Pass หรือ 'ผ่าน')"
"- ถ้าคะแนน (score) น้อยกว่า 50 → ให้ส่งเลข 0 กลับไป (เป็นตัวแทนของคำว่า Fail หรือ 'ตก')"

"เอ๊ะ... น้องๆ อาจจะสงสัยว่า ทำไมต้องส่งค่าเป็นเลข '1' กับเลข '0' ด้วยล่ะ? ทำไมไม่ return เป็นคำว่า 'True' หรือ 'False' แบบภาษาอื่น?"
"นี่คือความคลาสสิกของภาษา C ครับ ภาษา C สมัยดั้งเดิม 'ไม่มี' สิ่งที่เรียกว่า Boolean (True/False) ตรงๆ แบบภาษา Python หรือ Java นะครับ! กฎกติกาของภาษา C คือมันใช้เลข **1 (หรือเลขอะไรก็ได้ที่ไม่ใช่ศูนย์) เพื่อเป็นตัวแทนของคำว่า True (ความจริง)** และใช้เลข **0 เป็นตัวแทนหนึ่งเดียวของคำว่า False (เป็นเท็จ)** ครับ! นี่คือความรู้ระดับลึกที่โปรแกรมเมอร์ภาษา C ต้องรู้เลยนะครับ!"

"เรามาดูโค้ดการรวมร่างของฟังก์ชันกับ if-else กันครับ:"
```c
#include <stdio.h>

// สร้างฟังก์ชันครูตรวจข้อสอบ
int check_pass(int score) {
    if (score >= 50) {
        return 1;  // คืนค่า 1 แปลว่า 'ผ่าน (True)'
    } else {
        return 0;  // คืนค่า 0 แปลว่า 'ตก (False)'
    }
}

int main() {
    int score;
    printf("Enter exam score: ");
    scanf("%d", &score);

    // ส่งคะแนนไปให้ครูตรวจ แล้วรอรับผลการตัดสิน (0 หรือ 1)
    int result = check_pass(score);

    // เอาผลการตัดสินมาเช็คดูอีกทีว่า ผ่านหรือไม่ผ่าน
    if (result == 1) {
        printf("PASSED! ยินดีด้วยคุณสอบผ่าน!\n");
    } else {
        printf("FAILED! เสียใจด้วยคุณสอบตก!\n");
    }
    return 0;
}
```

"ลองทดสอบลอจิกกันดู:"
"- ถ้าเราป้อนคะแนน 70 → โปรแกรมเรียก `check_pass(70)` → ฟังก์ชันเช็คว่า 70 >= 50 ใช่ไหม? ใช่! → ฟังก์ชันจะดีดตัวเลข 1 กลับมา → `result` กลายเป็น 1 → เข้าเงื่อนไข `if (result == 1)` → สั่งพิมพ์ข้อความ 'PASSED!'"
"- ถ้าเราป้อนคะแนน 35 → โปรแกรมเรียก `check_pass(35)` → ฟังก์ชันเช็คว่า 35 >= 50 ใช่ไหม? ไม่ใช่! → ตกไปที่ else และดีดตัวเลข 0 กลับมา → `result` กลายเป็น 0 → เข้าเงื่อนไข else → สั่งพิมพ์ข้อความ 'FAILED!'"

"ข้อนี้สอนสัจธรรมข้อใหญ่ให้เราเลยครับว่า 'ฟังก์ชันไม่ได้มีหน้าที่แค่เอาตัวเลขมาบวกลบคูณหารเท่านั้น' แต่มันยังสามารถ **ใช้ลอจิกช่วยเราตัดสินใจ** ได้ด้วย! เราเอาความยุ่งยากของการเช็คเงื่อนไขไปหมกซ่อนไว้ในฟังก์ชัน เวลาเราใช้งานใน main เราก็แค่ส่งคะแนนไป แล้วรอรับคำตอบ 0 กับ 1 ชีวิตใน main ก็จะสะอาดและสบายขึ้นเยอะเลยครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **แผนภาพลอจิก (Flow Diagram):** score >= 50 → return 1 (Pass), score < 50 → return 0 (Fail)
- **โค้ดเต็ม + Output ทดสอบ** (ทดสอบทั้งเคสผ่านและเคสตก)
- **Key Insight (จุดประกายความรู้ระดับโปร):** แว่นขยายส่องไปที่เลข 1 และ 0 อธิบายว่า ในระบบของภาษา C, ค่า **1 = True (ความจริง)** และ **0 = False (เป็นเท็จ)**
