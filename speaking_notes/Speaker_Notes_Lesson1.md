## 📄 หน้าที่ 2: พี่อยากให้ทุกคนสัญญากับพี่ 3 ข้อนะครับ

### 🎤 บทพูด:
> พี่อยากให้ทุกคนสัญญากับพี่ 3 ข้อนะครับ:
> 
> ข้อ 1 — **อย่ากลัว Error** ครับ Error ไม่ใช่ศัตรู Error คือเพื่อนที่จะบอกเราว่าตรงไหนผิด เหมือนครูตรวจการบ้านให้ฟรีเลย
> 
> ข้อ 2 — **กล้าถามเสมอ** ครับ ถามโง่ดีกว่าโง่แล้วไม่ถาม ไม่มีคำถามไหนที่น่าอายทั้งนั้นครับ
> 
> ข้อ 3 — **พิมพ์ตามพี่** ครับ อย่านั่งดูอย่างเดียว ให้ลงมือพิมพ์ตาม เพราะการเขียนโค้ดมันเหมือนการว่ายน้ำครับ อ่านหนังสือสอนว่ายน้ำ 100 เล่มก็ว่ายไม่ได้ ต้องกระโดดลงน้ำจริงๆ ถึงจะเป็น!
> 
> ทุกคนพร้อมไหมครับ? … (รอเสียงตอบรับ) … เยี่ยมเลย! งั้นเราไปเริ่มกันเลยครับ!

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- สัญญา 3 ข้อ:
  1. ❌ อย่ากลัว Error
  2. 🙋‍♂️ กล้าถามเสมอ
  3. 💻 พิมพ์ตามพี่

## 📄 หน้าที่ 3: Topic Overview — ภาพรวม 7 หัวข้อที่จะเรียน

### 🎤 บทพูด:
> มาดูกันครับว่าวันนี้เราจะเรียนอะไรกันบ้าง บนสไลด์จะเห็นว่ามี 7 หัวข้อใหญ่เลยนะครับ แต่อย่าเพิ่งตกใจนะ! ฟังพี่อธิบายก่อน
> 
> **หัวข้อที่ 1** คือ "แนะนำภาษาซีเบื้องต้น" — พี่จะพาน้องรู้จักว่าภาษา C คืออะไร ทำไมเราถึงเรียนมัน แล้วก็เขียนโปรแกรมแรกกันเลยครับ
> **หัวข้อที่ 2** คือ "ชนิดของข้อมูลและกฎการตั้งชื่อ" — ง่ายๆ เลยครับ มันก็คือเรื่องของกล่องใส่ของ ว่ากล่องแต่ละใบมันใส่อะไรได้บ้าง แล้วเราจะตั้งชื่อกล่องมันยังไง
> **หัวข้อที่ 3** คือ "การดำเนินการทางคณิตศาสตร์และตรรกศาสตร์" — คือเรื่องบวก ลบ คูณ หาร 
> **หัวข้อที่ 4** คือ "การแสดงผลและการรับค่าพื้นฐาน" — พูดง่ายๆ คือการสอนให้คอมพิวเตอร์ "พูด" ออกหน้าจอ และ "ฟัง" ข้อมูลที่เราพิมพ์เข้าไป
> **หัวข้อที่ 5** คือ "คำสั่งควบคุมและคำสั่งวนซ้ำ" — อันนี้เริ่มสนุกแล้วครับ! มันคือการสอนให้คอมพิวเตอร์ "ตัดสินใจ" ได้ 
> **หัวข้อที่ 6** คือ "อาร์เรย์" — มันคือวิธีเก็บข้อมูลเป็นชุดๆ 
> **หัวข้อที่ 7** คือ "User-Define Function" — คือการสร้างคำสั่งขึ้นมาใช้เอง 
> 
> ฟังดูเยอะใช่ไหมครับ? แต่ไม่ต้องห่วงเลย พี่จะพาน้องๆ ไปทีละสเต็ป ค่อยๆ ก้าว และมีแบบฝึกหัดให้ลงมือทำจริงครับ!

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- **ตาราง 7 Topics:**
  1. Introduction to C
  2. Data Types & Naming Rules
  3. Math & Logic Operations
  4. Input / Output
  5. Flow Control & Loops
  6. Arrays
  7. User-Defined Functions

## 📄 หน้าที่ 5: คอมพิวเตอร์มันไม่เข้าใจภาษามนุษย์

### 🎤 บทพูด:
> ก่อนอื่นพี่อยากให้ทุกคนนึกภาพนะครับ สมมติเราอยากสั่งให้คอมพิวเตอร์ทำอะไรสักอย่าง เช่น อยากให้มันแสดงข้อความว่า "สวัสดี" บนหน้าจอ ปัญหาคือ... คอมพิวเตอร์มันไม่เข้าใจภาษามนุษย์ครับ! มันเข้าใจแค่เลข 0 กับ 1 — หรือที่เรียกว่าเลขฐานสอง เปิดไฟ ปิดไฟ แค่นั้นเอง!

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- รูปคนพูด "สวัสดี!"
- รูปลูกศรชี้ไปที่คอมพิวเตอร์ที่งง และมีเลข `01010101` ลอยอยู่
- ข้อความ: **คอมพิวเตอร์เข้าใจแค่ 0 กับ 1 (Machine Language)**

### 💡 เทคนิคการสอน:
- ทำเสียงหุ่นยนต์ตลกๆ ตอนพูดถึง 0 กับ 1 เพื่อเรียกรอยยิ้ม และให้เด็กจดจำง่ายขึ้น

## 📄 หน้าที่ 6: ภาษาโปรแกรมคือสะพานเชื่อม

### 🎤 บทพูด:
> เราถึงต้องมี "ภาษาโปรแกรม" มาเป็นสะพานเชื่อมครับ! ให้มนุษย์เขียนคำสั่งในรูปแบบภาษาอังกฤษที่พอจะอ่านออก แล้วค่อยให้ "เครื่องมือแปลภาษา" มาแปลงข้อความพวกนั้น เป็น 0 กับ 1 ให้คอมพิวเตอร์เข้าใจอีกทีนึง ภาษา C ก็คือหนึ่งในสะพานเชื่อมที่เก่งมากๆ เลยครับ

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- แผนภาพ: **มนุษย์ (Code)** ➡️ **ภาษาโปรแกรม (สะพาน)** ➡️ **คอมพิวเตอร์ (0101)**
- ระบุคำว่า: Machine Language / Assembly Language

### 💡 เทคนิคการสอน:
- เน้นคำว่า "สะพานเชื่อม" ให้เห็นภาพว่าเราไม่ได้คุยกับเครื่องตรงๆ

## 📄 หน้าที่ 7: ประวัติของภาษา C

### 🎤 บทพูด:
> **ภาษา C** เป็นหนึ่งในภาษาโปรแกรมที่เก่าแก่ที่สุดและทรงพลังที่สุดเท่าที่เคยมีมาเลยครับ มันถูกสร้างขึ้นโดยคุณลุง **Dennis Ritchie** — จำชื่อนี้ไว้นะครับ — ในช่วง **ยุค 1970** ที่ Bell Labs สหรัฐอเมริกา
> 
> ตอนนั้นคุณลุง Dennis สร้างภาษา C ขึ้นมาเพื่อจุดประสงค์เดียวเลยครับ คือ **สร้างระบบปฏิบัติการ UNIX** ซึ่ง UNIX เนี่ย ถ้าน้องๆ เคยได้ยินคำว่า Linux, macOS หรือแม้กระทั่ง Android ในมือถือเรา มันล้วนแล้วแต่มีรากฐานมาจาก UNIX ทั้งนั้นเลยครับ! 

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- รูปภาพ Dennis Ritchie (ภาพขาวดำ)
- ข้อความ: ถูกพัฒนาโดย **Dennis Ritchie** ในช่วงปี **1970**
- จุดประสงค์แรก: **สร้างและพัฒนาระบบปฏิบัติการ UNIX**

### 💡 เทคนิคการสอน:
- เล่าเป็นนิทานสั้นๆ ให้ดูน่าสนใจ ไม่ท่องจำ
- **เกร็ดเสริม** — "คุณลุง Dennis Ritchie ได้รับรางวัล Turing Award ซึ่งเป็นรางวัลระดับโนเบลของวงการคอมพิวเตอร์เลยนะครับ"

## 📄 หน้าที่ 8: ภาษาคอมพิวเตอร์เชิงโครงสร้าง

### 🎤 บทพูด:
> ภาษา C เป็น **"ภาษาคอมพิวเตอร์เชิงโครงสร้าง" (Structural Language)** คำว่า "เชิงโครงสร้าง" หมายความว่า เราเขียนโค้ดเป็นบล็อกๆ เป็นก้อนๆ อย่างเป็นระเบียบ คิดง่ายๆ เหมือนการสร้างตึกครับ เราก็ก่อมันขึ้นมาทีละชั้น ทีละห้อง ไม่ใช่สร้างแบบมัวๆ สุ่มสี่สุ่มห้า
> 
> พี่รู้ว่าหลายคนอาจจะคิดว่า "เฮ้ย ภาษาที่เกิดมาตั้งแต่ปี 1970 มันจะไม่ล้าสมัยหรอ? ทำไมไม่เรียน Python ที่ฮิตๆ ไปเลย?" … คำตอบอยู่ในสไลด์หน้าถัดไปครับ!

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- ภาษาคอมพิวเตอร์เชิงโครงสร้าง (**Structural Language**)
- รูปการต่อเลโก้ หรือการสร้างตึกเป็นชั้นๆ

### 💡 เทคนิคการสอน:
- **ใช้อุปมา "ภาษาแม่"** — "ภาษา C เหมือนภาษาไทยครับ ถ้าเราอ่านออกเขียนได้ภาษาไทย ไปเรียนภาษาลาว ภาษาเขมร ก็จะง่ายมากเพราะมีรากคล้ายกัน"

## 📄 หน้าที่ 10: ทำไมต้องเรียน C? เหตุผลที่ 1

### 🎤 บทพูด:
> เหตุผลที่ 1 เลยครับ — **ภาษา C เป็นพื้นฐานของทุกอย่าง!** ภาษาดังๆ ที่น้องอาจเคยได้ยิน อย่าง C++, Java, C# มันล้วนแล้วแต่เกิดมาจากรากของภาษา C ทั้งนั้นเลยครับ ถ้าน้องเข้าใจ C แล้ว การไปเรียน Java หรือ Python ต่อจะง่ายมาก! 
> 
> ลองคิดดูนะครับ — มันเหมือนเราเรียนขับรถเกียร์ธรรมดาก่อน พอเป็นแล้ว ไปขับรถเกียร์ออโต้มันง่ายมาก! ภาษา C ก็เหมือนเกียร์ธรรมดาแหละครับ — มันสอนให้เราเข้าใจ "กลไกข้างในจริงๆ" ของคอมพิวเตอร์

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- **ภาษา C คือรากฐาน (Root Language)**
- แผนผัง: C ➡️ C++ ➡️ Java / C#
- ชี้ให้เห็นว่า Python (CPython) ก็เขียนด้วย C

### 💡 เทคนิคการสอน:
- **อุปมา "เกียร์ธรรมดา vs ออโต้"** ช่วยได้มากกับเด็กมัธยม ให้รู้ว่ายอมเหนื่อยหน่อยตอนแรก สบายตอนหลัง

## 📄 หน้าที่ 12: เทียบ Syntax

### 🎤 บทพูด:

## 📄 หน้าที่ 16: โครงสร้างพื้นฐานของโปรแกรมภาษา C

### 🎤 บทพูด:
> มาถึงส่วนสำคัญที่สุดแล้วครับ — **"โครงสร้างพื้นฐาน"** นี่คือ "โครงกระดูก" ของโปรแกรม C ทุกตัวเลย! พี่จะอธิบาย 3 ส่วนหลักนะครับ:
> 
> **ส่วนที่ 1: `#include <stdio.h>`**
> นี่คือ **ส่วน Header** เป็นการบอกว่า "ฉันจะขอใช้เครื่องมือเสริมนะ!" `stdio.h` คือเครื่องมือรับ/แสดงผล เหมือนเราจะทำเค้กก็ต้องเอาแป้งมาเตรียมไว้ก่อน ถ้าไม่มีบรรทัดนี้ เราจะสั่งพิมพ์ข้อความไม่ได้เลยครับ!
> 
> **ส่วนที่ 2: `int main() { }`**
> ตรงนี้คือ **ฟังก์ชันหลัก** เป็น **"ประตูหน้าบ้าน"** ของโปรแกรม! เวลาคอมพิวเตอร์รันโปรแกรม มันจะวิ่งเข้าประตูนี้ก่อนเสมอ โค้ดทั้งหมดของเราต้องเขียนอยู่ในปีกกา `{ }` คู่นี้นะครับ
> 
> **ส่วนที่ 3: `return 0;`**
> บรรทัดสุดท้ายก่อนปิดปีกกา เป็นการบอกระบบว่า **"งานเสร็จสมบูรณ์ ไม่มี Error"** (0 คือปกติดี)
>
> จำ 3 ส่วนนี้ไว้นะครับ นี่คือโครงร่างที่เราต้องพิมพ์เสมอ!

### 📊 สิ่งที่ต้องแสดงในสไลด์:
```c
#include <stdio.h>  // ← ส่วน Header: เตรียมเครื่องมือ

int main() {        // ← main function: ประตูหน้าบ้าน

    // โค้ดของเราจะอยู่ตรงนี้ 

    return 0;       // ← จบการทำงานแบบปกติ
}
```

### 💡 เทคนิคการสอน:
- **ชี้ทีละส่วน** — ใช้ Laser pointer ชี้ทีละบรรทัดขณะอธิบาย
- ย้ำเรื่อง case-sensitive "ตัวพิมพ์เล็กใหญ่มีผลนะครับ พิมพ์ `main` เป็นตัวเล็กทั้งหมด!"

## 📄 หน้าที่ 17: โปรแกรมแรก / การทำงาน — Hello World!

### 🎤 บทพูด:
> มาเขียน **โปรแกรมแรก** กันเลย! ในวงการโปรแกรมเมอร์มีธรรมเนียมว่า โปรแกรมแรกในชีวิตต้องเป็นการสั่งพิมพ์คำว่า **"Hello World!"** ครับ
> 
> ดูโค้ดบนจอนะครับ โครงสร้างเหมือนเดิมเป๊ะ แต่เราแทรก `printf("hello world!");` เข้าไปตรงกลาง
> กฎของ `printf` คือ ข้อความต้องครอบด้วยเครื่องหมายคำพูด `" "` และจบด้วยเซมิโคลอน `;` เสมอ!
> 
> อ้อ เห็นเครื่องหมาย `//` ไหมครับ? นั่นคือ **"Comment"** คอมพิวเตอร์จะไม่อ่านบรรทัดนี้ เราเอาไว้เขียนจดโน้ตให้ตัวเองครับ
> 
> **(สาธิตบนจอ)** มาลองกันครับ! พิมพ์ตามพี่เลย แล้วกดปุ่ม Run... เห็นข้อความออกทางขวาไหมครับ? ยินดีด้วย! ทุกคนเป็นโปรแกรมเมอร์แล้ว ปรบมือให้ตัวเองหน่อยครับ! 👏

### 📊 สิ่งที่ต้องแสดงในสไลด์:
```c
#include <stdio.h>
// นี่คือ Comment — คอมพิวเตอร์จะไม่อ่าน

int main() {
    printf("hello world!");
    return 0;
}
```
**ผลลัพธ์ (Output):** `hello world!`

### 💡 เทคนิคการสอน:
- **ต้องสาธิตสด!** พิมพ์ให้เด็กดูช้าๆ
- ปรบมือสร้างความภาคภูมิใจตอนที่เด็กรันผ่าน

## 📄 หน้าที่ 19: ข้อควรระวัง (Syntax Errors ที่พบบ่อย)

### 🎤 บทพูด:
> ก่อนไปต่อ พี่ขอเตือน 2 เรื่องสำคัญที่ทำให้โปรแกรม Error บ่อยที่สุดครับ!
> 
> **ข้อ 1: ในการเขียนฟังก์ชั่นถ้ามีวงเล็บเปิดต้องมีวงเล็บปิดให้ครบคู่เสมอ!** ไม่ว่าจะเป็นวงเล็บ `<>`, `()`, `{}`, `[]` เทคนิคคือ พิมพ์เปิดปุ๊บ ให้พิมพ์ปิดดักไว้เลย แล้วค่อยเลื่อนเมาส์กลับมาพิมพ์ตรงกลางครับ จะไม่มีทางลืมแน่นอน!
> 
> **ข้อ 2: ในการทำงาน 1 บรรทัดถ้าไม่ใช่ฟังก์ชันหรือคำสั่งที่ขึ้นต้นด้วย # ต้องลงท้ายด้วย `;` เสมอ!** เซมิโคลอนเหมือนเครื่องหมาย "จุด (Full stop)" ท้ายประโยคภาษาอังกฤษครับ 
> 
> ถ้าลืม `;` โปรแกรมจะพังทันที! ลองดูบนจอพี่นะครับ (ลบ ; แล้วกด Run โชว์ Error) นี่ไง หน้าแดงเลย 555

### 📊 สิ่งที่ต้องแสดงในสไลด์:
**⚠️ ข้อควรระวัง (Syntax Rules)**
- **กฎข้อ 1:** ในการเขียนฟังก์ชั่นถ้ามีวงเล็บเปิดต้องมีวงเล็บปิดให้ครบคู่เสมอ `<>`, `()`, `{}`, `[]`
- **กฎข้อ 2:** ในการทำงาน 1 บรรทัดถ้าไม่ใช่ฟังก์ชันหรือคำสั่งที่ขึ้นต้นด้วย `#` ต้องลงท้ายด้วย `;` เสมอ

### 💡 เทคนิคการสอน:
- สาธิตลืม `;` สดๆ ให้เด็กดูว่า Error หน้าตาเป็นยังไง และสอนวิธีอ่านบรรทัดที่มี Error

## 📄 หน้าที่ 21: ตัวแปร (Variables) — คืออะไร?

### 🎤 บทพูด:
> "ตัวแปร" คืออะไร? พี่อยากให้ทุกคนนึกภาพง่ายๆ ว่ามันคือ **"กล่องพัสดุ"** ครับ
> 
> สมมติเราอยากเก็บของ เราก็ต้องหากล่องมา 1 ใบ แล้วเราต้องทำ 2 อย่าง:
> 1. เลือกว่าจะเอากล่องแบบไหน (กล่องใส่แก้วน้ำ กล่องใส่เสื้อผ้า)
> 2. เขียนป้ายชื่อแปะหน้ากล่อง จะได้ไม่ลืมว่าข้างในคืออะไร
> 
> ตัวแปรในภาษา C ก็เหมือนกันเป๊ะครับ! มันคือกล่องใน Memory (RAM) ของเครื่อง เราต้องบอก **"ประเภทของกล่อง" (Data Type)** และติด **"ป้ายชื่อ" (Variable Name)** ให้มันครับ

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- รูปภาพ: กล่องพัสดุเปล่าๆ และมีมือจับปากกาเมจิกเขียนป้ายชื่อแปะกล่อง
- ตัวแปร (Variable) = กล่องสำหรับเก็บข้อมูลในคอมพิวเตอร์

### 💡 เทคนิคการสอน:
- หากล่องกระดาษของจริงมาถือประกอบการสอน จะทำให้เด็กมัธยมหรือประถมเห็นภาพชัดขึ้นมากๆ

## 📄 หน้าที่ 22: การตั้งชื่อตัวแปร (การประกาศและชนิดข้อมูล)

### 🎤 บทพูด:
> การสร้างกล่องในภาษา C จะเขียนแบบนี้ครับ:
> **`ชนิดข้อมูล  ชื่อตัวแปร;`** เช่น **`int num;`** แปลว่า "สร้างกล่องประเภท int แล้วตั้งชื่อว่า num"
> 
> แล้วชนิดข้อมูลมีอะไรบ้าง? หลักๆ ที่ใช้บ่อยมี 3 ชนิดครับ:
> 1. **`int`** (Integer) — กล่องใส่ **จำนวนเต็ม** เช่น 5, -10 กล่องนี้ใส่ทศนิยมไม่ได้นะครับ เหมือนกระปุกหมูที่หยอดได้แต่เหรียญ
> 2. **`float`** — กล่องใส่ **ทศนิยม** เช่น 3.14, 99.9 เหมือนถ้วยตวงของเหลว
> 3. **`char`** (Character) — กล่องจิ๋ว ใส่ **ตัวอักษรได้แค่ 1 ตัว** เท่านั้น เช่น 'A', 'z' เหมือนกรอบรูปเล็กๆ
>
> คอมพิวเตอร์มันมองทุกอย่างเป็น 0 กับ 1 เราเลยต้องกำหนดประเภทให้มันรู้ว่า "อ๋อ นี่คือตัวเลขนะ ไม่ใช่ตัวอักษร!"

### 📊 สิ่งที่ต้องแสดงในสไลด์:
**รูปแบบ:** `ชนิดข้อมูล  ชื่อตัวแปร;` (เช่น `int num;`)
- **`int`**: จำนวนเต็ม (5, -10, 100)
- **`float`**: ทศนิยม (3.14, 0.5)
- **`char`**: ตัวอักษร 1 ตัว ('A', 'c')

### 💡 เทคนิคการสอน:
- ถามเด็ก: "ถ้าพี่จะเก็บค่าเกรด A B C ควรใช้กล่องอะไรดีครับ?" ให้เด็กตอบพร้อมกัน (char)

### 🎤 บทพูด:
> ชนิดแรกที่ใช้บ่อยสุดคือ **`int` (จำนวนเต็ม)** ครับ 
> int ใช้เก็บตัวเลขที่ไม่มีทศนิยม เช่น 5, -10 แต่รู้ไหมครับว่ากล่อง int ก็มีไซส์ย่อยๆ ด้วยนะ!

### 💡 เทคนิคการสอน:
- ย้ำชัดๆ ว่า Single Quote สำหรับ 1 ตัวอักษร, Double Quote สำหรับข้อความยาวๆ

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- ตาราง Format Specifiers:
  - `%d` ➡️ int (จำนวนเต็ม)
  - `%f` ➡️ float (ทศนิยม)
  - `%c` ➡️ char (ตัวอักษร)

### 💡 เทคนิคการสอน:
- เน้นย้ำว่า "รหัสเจาะรู ต้องตรงกับ ประเภทกล่อง เสมอ!"

## 📄 หน้าที่ 28.3: การขึ้นบรรทัดใหม่ (\n)

### 🎤 บทพูด:
> และสิ่งที่คู่กับ printf เสมอ คือ **`\n` (backslash n)** ครับ!
> มันคือสัญลักษณ์สั่ง **"ขึ้นบรรทัดใหม่"** เหมือนเวลาเรากดปุ่ม Enter บนคีย์บอร์ด 
> 
> ถ้าเราเขียน `printf("Hello"); printf("World");` แบบไม่มี `\n` คอมพิวเตอร์มันจะพิมพ์ติดกันเป็น `HelloWorld` ยาวเฟื้อยเลย! ดังนั้นถ้าอยากให้สวย ต้องใส่ `\n` เข้าไปท้ายประโยคด้วยนะครับ

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- อธิบาย `\n` (Newline) = ปุ่ม Enter
- ตัวอย่างเปรียบเทียบ: มี `\n` vs ไม่มี `\n`

### 💡 เทคนิคการสอน:
- ย้ำว่าต้องเป็น Backslash `\` นะ ไม่ใช่ Slash `/` เด็กชอบพิมพ์ผิดบ่อยมาก

## 📄 หน้าที่ 29.1: การแสดงผลตัวแปร — ตัวอย่างโค้ด (printf in action)

### 🎤 บทพูด:
> มาดูตัวอย่างโค้ดจริงๆ กันครับ บนจอพี่สร้างตัวแปรไว้ 3 ตัว
> `score` เก็บ 100, `gpa` เก็บ 3.99, `grade` เก็บ 'A'
> 
> เวลาจะปรินต์พี่ก็เขียนแบบนี้ครับ:
> `printf("Score: %d\n", score);` 
> เห็นไหมครับ พี่เจาะรู `%d` ไว้หลังคำว่า Score: แล้วเอาตัวแปร `score` ไปรอต่อคิวอยู่หลังเครื่องหมายจุลภาค 
> พอมันทำงานปุ๊บ มันก็จะดูดเอา 100 เข้าไปเสียบตรง `%d` พอดีเป๊ะ!

### 📊 สิ่งที่ต้องแสดงในสไลด์:
```c
int score = 100;
float gpa = 3.99;
char grade = 'A';

printf("Score: %d\n", score);
printf("GPA: %f\n", gpa);
printf("Grade: %c\n", grade);
```

### 💡 เทคนิคการสอน:
- ทำอนิเมชั่นในสไลด์ (ถ้าทำได้) ให้ลูกศรชี้ว่าตัวแปรด้านหลัง ลอยเข้าไปเสียบแทนที่ตำแหน่งของ `%` ในข้อความ

## 📄 หน้าที่ 29.2: เทคนิคการจัดรูปแบบทศนิยม (%.2f)

### 🎤 บทพูด:
> มีความลับสวรรค์อย่างนึงของ `%f` ครับ ปกติถ้าเราสั่ง `%f` คอมพิวเตอร์มันจะใจดีเกินไป แสดงทศนิยมมาให้ถึง 6 ตำแหน่ง! เช่น 3.990000 ซึ่งมันรกรุงรังมาก
> 
> ถ้าเราอยากให้มันแสดงแค่ 2 ตำแหน่ง ให้เราแทรกจุดสองเข้าไปตรงกลางครับ เป็น **`%.2f`** แค่นี้เลย!
> ถ้าอยากได้ 1 ตำแหน่งก็ **`%.1f`** ง่ายไหมครับ? เทคนิคนี้ใช้บ่อยมากเวลาเขียนโปรแกรมคำนวณเงินครับ

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- เทคนิคพิเศษ: การจำกัดจุดทศนิยม
- `printf("%.2f", gpa);` ➡️ แสดงผล 3.99 (ตัดศูนย์ด้านหลังทิ้ง)

### 💡 เทคนิคการสอน:
- ให้เด็กเปิด Programiz ขึ้นมาลองพิมพ์ `%f` เทียบกับ `%.2f` ดูด้วยตาตัวเอง

## 📄 หน้าที่ 30.1: คำสั่ง scanf (หูของโปรแกรม)

### 🎤 บทพูด:
> เอาล่ะ มีปากแล้ว ต่อไปก็ต้องมีหูรับฟังผู้ใช้บ้างครับ!
> คำสั่งรับข้อมูลที่เราพิมพ์ผ่านคีย์บอร์ด คือ **`scanf`** ครับ ย่อมาจาก Scan Formatted
> 
> รูปแบบมันคล้าย printf เลยครับ คือ **`scanf("%_", &ชื่อตัวแปร);`** 
> เราเจาะรูบอกมันว่า "รอรับเลขจำนวนเต็มนะ (%d)" พอผู้ใช้พิมพ์เลขลงไป กด Enter ปุ๊บ มันก็จะเอาค่านั้นไปเก็บไว้ในตัวแปรทันที!

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- หัวข้อ: **คำสั่ง scanf (รับค่าจากผู้ใช้)**
- รูปแบบ: `scanf("%_", &ชื่อข้อมูล);`
- Format Specifiers เหมือน printf: `%d`, `%f`, `%c`

### 💡 เทคนิคการสอน:
- สาธิตว่าเมื่อโปรแกรมเจอ scanf มันจะ "หยุดกึก" เพื่อรอให้เราพิมพ์ โปรแกรมจะยังไม่ทำงานต่อจนกว่าเราจะกด Enter

## 📄 หน้าที่ 30.2: กฎเหล็กของ scanf (เครื่องหมาย &)

### 🎤 บทพูด:
> แต่ช้าก่อน! ทุกคนเห็นเครื่องหมาย **`&` (แอมเปอร์แซนด์)** หน้าชื่อตัวแปรไหมครับ?
> นี่คือ **กฎเหล็ก** และเป็นจุดที่คนตายกันเยอะที่สุด! ลืมกันบ่อยมาก!
> 
> เครื่องหมาย `&` แปลว่า **"ที่อยู่ของ (Address of)"** ครับ
> นึกภาพว่า `scanf` คือพนักงานส่งของ Shopee ครับ พนักงานมีกล่องพัสดุ(ข้อมูลที่เราพิมพ์) แต่พนักงานไม่รู้ว่าจะเอาไปส่งที่บ้านไหนในคอมพิวเตอร์! 
> เราเลยต้องติด `&` ไว้หน้าชื่อตัวแปร เพื่อบอกพนักงานว่า "ที่อยู่ของบ้าน age อยู่ตรงนี้นะ ส่งมาที่นี่เลย!"
> ถ้าเราลืม `&` ข้อมูลจะส่งไม่ถึง โปรแกรมก็จะ Error พังทลายเลยครับ!

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- ⚠️ **กฎเหล็ก: ห้ามลืม `&` หน้าตัวแปรเด็ดขาด!**
- ภาพ: พนักงาน Shopee/ไปรษณีย์ ยืนงงเพราะไม่มีที่อยู่บ้าน (Address)

### 💡 เทคนิคการสอน:
- เล่นคำให้จำง่าย: "`scanf` ต้องมี **แอมเปอร์แซนด์** เหมือน **แอมบูแลนซ์** ถ้าลืมใส่ โปรแกรมจะเข้าโรงบาล!"

## 📄 หน้าที่ 30.3: ตัวอย่างโค้ด scanf ในโลกจริง

### 🎤 บทพูด:
> มาดูการใช้ scanf แบบคอมโบต่อเนื่องครับ! 
> โปรแกรมมักจะเป็น Pattern นี้เสมอ: **สร้างกล่อง ➡️ ปรินต์ถาม ➡️ สแกนรับ ➡️ ปรินต์สรุป**
> 
> ดูโค้ดนี้นะครับ:
> `int age;` (สร้างกล่อง)
> `printf("How old are you?: ");` (ถามผู้ใช้)
> `scanf("%d", &age);` (หูรอฟัง และเอาไปส่งที่บ้าน &age)
> `printf("You are %d years old!", age);` (สรุปผล)
> 
> แล้วถ้าเรารับตัวเลข 2 ตัวล่ะ? พี่ก็ทำเหมือนเดิมครับ ปรินต์ถาม สแกนตัวที่ 1... แล้วก็ ปรินต์ถาม สแกนตัวที่ 2... จากนั้นเอามาบวกกัน แล้วปรินต์ผลลัพธ์ออกจอ ง่ายไหมครับ!

### 📊 สิ่งที่ต้องแสดงในสไลด์:
- โค้ดตัวอย่างที่ 1: รับอายุ
- โค้ดตัวอย่างที่ 2: รับตัวเลข 2 ตัวมาบวกกัน (เน้น `num1 + num2`)
- Flow: สร้าง ➡️ ถาม ➡️ รอรับ ➡️ สรุป

### 💡 เทคนิคการสอน:
- ให้เด็กลองเขียนโปรแกรมบวกเลขด้วยตัวเองบน Programiz โดยใช้ Pattern ที่สอนไป ถือเป็นการทบทวนก่อนจบบทเรียน

**🎤 บทพูด:**
