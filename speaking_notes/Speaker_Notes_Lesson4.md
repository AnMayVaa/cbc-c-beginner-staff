## 🏆 Workshop 1: เครื่องคิดเลขจิ๋ว (Mini Calculator)

### Slide 1.1: วิเคราะห์โจทย์และ Algorithm
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"ข้อนี้เป็นข้อแรกของ Workshop ครับ น้องๆ ทุกคน! มาวิเคราะห์กันก่อน — โจทย์ต้องการให้เราทำเครื่องคิดเลขที่รับตัวเลข 2 ตัว กับเครื่องหมาย 1 ตัว แล้วทำการคำนวณ"
"พี่ขอย้ำนะครับ อย่าเพิ่งรีบเปิดโปรแกรมพิมพ์โค้ด! เรามาคิดบนกระดาษกันทีละขั้นตอนก่อน"
"**Step 1:** การรับค่า — โปรแกรมของเราต้องรับอะไรบ้างเพื่อให้มันทำงานได้?
- แน่นอนครับ ต้องมีตัวเลขตัวแรก (เราจะใช้ประเภทข้อมูล int หรือ float ดี? พี่แนะนำ float ครับ จะได้รองรับทศนิยม)
- จากนั้นคือ เครื่องหมาย (ตัวนี้เป็นอักขระเดี่ยวๆ ต้องใช้ char นะครับ!)
- แล้วก็ตามด้วยตัวเลขตัวที่สอง (เป็น float เหมือนกัน)"

"**Step 2:** การตัดสินใจ — เครื่องหมายที่รับมาเป็นอะไร? เราจะใช้ if-else if ในการตรวจสอบครับ
- ถ้าเครื่องหมาย (op) == '+' → เราก็จับมันมาบวกกัน
- ถ้า op == '-' → เอามาลบกัน
- ถ้า op == '*' → เอามาคูณกัน
- ถ้า op == '/' → เอามาหารกัน
- แต่ถ้าไม่ใช่อะไรเลย (else) → ต้องแจ้งเตือนว่าเครื่องหมายไม่ถูกต้อง!"

"**Step 3:** แสดงผลลัพธ์ออกทางหน้าจอครับ"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โจทย์:** รับ 2 ตัวเลข + เครื่องหมาย (+, -, *, /) แล้วคำนวณ
- **Algorithm Flowchart:** รับ num1 → รับ op → รับ num2 → เช็คเงื่อนไข op ด้วย if-else → คำนวณ → แสดงผล
- เน้นภาพการคิดเป็น Step-by-Step

**💡 เทคนิคการสอน:**
- ถามนำเด็กๆ ว่า "ถ้าพี่อยากพิมพ์ 5.5 + 2 เราควรเก็บเลข 5.5 ไว้ในตัวแปรชนิดไหน?"
- ให้เด็กวาด Flowchart ตามก่อน

### Slide 1.2: ลุยเขียนโค้ดและข้อควรระวัง
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"ทีนี้เรามาดูโค้ดกันครับ มีจุดหนึ่งที่พี่อยากให้ทุกคนระวังมากๆ เรียกว่าเป็นกับดักของภาษา C เลย!"
"ตอนที่เรารับ char ด้วย scanf เราต้องเขียนว่า `scanf(" %c", &op);` น้องๆ สังเกตเห็นอะไรไหมครับ? พี่แอบใส่ **ช่องว่าง 1 ช่อง** ไว้หน้า `%c`! ทำไมต้องมีช่องว่าง?"
"เหตุผลคือ ตอนที่เราพิมพ์ตัวเลขตัวแรกเสร็จแล้วกดปุ่ม Enter... ตัว Enter (หรือ `\n`) มันจะตกค้างอยู่ในหน่วยความจำครับ ถ้าเราไม่ใส่ช่องว่าง scanf ตัวถัดไปมันจะไปคว้าเอา Enter มาเป็นเครื่องหมายแทน! ทำให้โปรแกรมข้ามการรับเครื่องหมายไปเลย ช่องว่างตัวนี้จะช่วย 'กิน' หรือเคลียร์ตัว Enter ที่ค้างอยู่ให้หมดไปก่อนครับ"

```c
#include <stdio.h>

int main() {
    float num1, num2;
    char op;

    printf("Enter number 1: ");
    scanf("%f", &num1);

    printf("Enter operator (+, -, *, /): ");
    scanf(" %c", &op);  // ข้อควรระวัง: ต้องมีช่องว่างหน้า %c!

    printf("Enter number 2: ");
    scanf("%f", &num2);

    if (op == '+') {
        printf("Result: %.2f\n", num1 + num2);
    } else if (op == '-') {
        printf("Result: %.2f\n", num1 - num2);
    } else if (op == '*') {
        printf("Result: %.2f\n", num1 * num2);
    } else if (op == '/') {
        if (num2 == 0) {
            printf("Error: Cannot divide by zero!\n");
        } else {
            printf("Result: %.2f\n", num1 / num2);
        }
    } else {
        printf("Invalid operator!\n");
    }
    return 0;
}
```

"และอีกจุดที่พี่แถมให้ สังเกตเงื่อนไขการหารนะครับ พี่เพิ่มการเช็คว่า num2 เป็น 0 หรือเปล่า เพราะในทางคณิตศาสตร์ เราหารด้วย 0 ไม่ได้! ถ้าไม่ดักเอาไว้ โปรแกรมอาจจะพัง หรือขึ้น Infinity นี่คือตัวอย่างของการเขียนโปรแกรมที่ 'คิดรอบด้าน' ครับ"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โค้ดเต็ม + Output ตัวอย่าง**
- **เตือนตัวใหญ่ๆ:** `scanf(" %c", &op);` ระวังช่องว่างหน้า `%c`!
- **Bonus:** การเช็คการหารด้วย 0 (Zero Division Error)

**💡 เทคนิคการสอน:**
- ให้เด็กลองเขียนแบบไม่มีช่องว่างหน้า `%c` ก่อน ให้เขาเจอบั๊กด้วยตัวเอง แล้วค่อยเฉลยวิธีแก้ เขาจะจำฝังใจ
- ถ้าเด็กคนไหนทำเสร็จเร็ว ท้าให้เขาเพิ่มเครื่องหมาย `%` (หารเอาเศษ) เข้าไปด้วย (คำใบ้: ต้องแปลงเป็น int ก่อน)

## 🏆 Workshop 2: ตู้กดน้ำอัตโนมัติ (Vending Machine)

### Slide 2.1: วิเคราะห์กระบวนการทำงาน
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"โจทย์ข้อนี้สนุกครับ เราจะมาสร้างตู้กดน้ำอัตโนมัติจำลองกัน! เวลาน้องๆ ไปกดน้ำที่ตู้ ตู้มันทำงานยังไงครับ?"
"1. ตู้มันจะโชว์เมนูก่อนใช่ไหม ว่ามีอะไรขายบ้าง เช่น น้ำเปล่า 10 บาท, โคล่า 15 บาท, ชาเขียว 20 บาท"
"2. จากนั้นมันจะรอให้เรากดเลือกเมนู"
"3. ระบบจะรับรู้ราคาของสินค้าที่เราเลือก"
"4. ตู้จะบอกให้เราหยอดเงิน"
"5. ทีนี้แหละสำคัญ! ตู้ต้องตัดสินใจ: ถ้าเงินที่เราหยอด มากกว่าหรือเท่ากับราคาน้ำ → มันจะทอนเงินให้เรา แต่ถ้าเงินไม่พอ → มันก็ต้องเตือนว่า 'เงินไม่พอนะ ขาดอีกเท่าไหร่'"
"นี่แหละครับ ลอจิกทั้งหมดที่เราจะเอาไปเขียนโปรแกรม!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โจทย์:** กำหนดราคาน้ำ ให้เลือกเมนู + หยอดเงิน เช็คยอดเงินเพื่อทอน หรือเตือนว่าเงินไม่พอ
- **Flowchart:** แสดงเมนู → เลือก → กำหนดราคาด้วย if-else → รับเงิน → เช็ค (เงิน >= ราคา) → ทอนเงิน / แจ้งเงินไม่พอ

**💡 เทคนิคการสอน:**
- ชวนเด็กๆ คุยเรื่องประสบการณ์กดตู้เต่าบิน หรือตู้กดน้ำอื่นๆ เพื่อให้เห็นภาพระบบจริง

### Slide 2.2: โค้ดและ Early Return
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"เอาล่ะ มาแปลงความคิดเป็นโค้ดกันครับ"

```c
#include <stdio.h>

int main() {
    int choice, money, price;

    printf("=== Vending Machine ===\n");
    printf("1. Water  (10 baht)\n");
    printf("2. Cola   (15 baht)\n");
    printf("3. Green Tea (20 baht)\n");
    printf("Choose menu: ");
    scanf("%d", &choice);

    // กำหนดราคาตามเมนูที่เลือก
    if (choice == 1) {
        price = 10;
    } else if (choice == 2) {
        price = 15;
    } else if (choice == 3) {
        price = 20;
    } else {
        printf("Invalid menu!\n");
        return 0;  // จบโปรแกรมทันที
    }

    printf("Insert money: ");
    scanf("%d", &money);

    // ตรวจสอบการทอนเงิน
    if (money >= price) {
        printf("Change: %d baht\n", money - price);
    } else {
        printf("Not enough! Need %d more baht\n", price - money);
    }
    return 0;
}
```
"พี่อยากให้ดูตรง `return 0;` ในเงื่อนไข else ตอนที่เลือกเมนูผิดนะครับ เทคนิคนี้เราเรียกว่า **Early Return** คือการสั่งจบโปรแกรมทันทีตั้งแต่เนิ่นๆ!"
"ทำไมต้องทำแบบนี้? เพราะถ้าผู้ใช้กดเลือกเมนูที่ 4 ซึ่งมันไม่มีในระบบ ก็ไม่มีประโยชน์อะไรที่เราจะไปถามเขาต่อว่า 'กรุณาหยอดเงิน' จริงไหมครับ? เราก็ตัดจบทันทีเลยดีกว่า"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โค้ดเต็ม + Output ตัวอย่าง** (ทั้งกรณีปกติ และกรณีเงินไม่พอ)
- **Highlight:** `return 0;` ตรงเงื่อนไข `Invalid menu!` และอธิบาย Early Return

**💡 เทคนิคการสอน:**
- ให้เด็กออกแบบเมนูน้ำหรือขนมของตัวเองเลย กำหนดราคาเองตามใจชอบ
- ท้าทายคนเก่ง: "ลองเพิ่มเมนูที่ 4 เข้าไปด้วยตัวเองสิ"

## 🏆 Workshop 3: เกมทายใจตัวเลข (Guess the Number)

### Slide 3.1: กติกาและการใช้ While Loop
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"มาถึงโจทย์ข้อที่ 3 ข้อนี้พี่บอกเลยว่าสนุกที่สุดครับ เพราะเรากำลังจะสร้าง **เกม!**"
"เกมนี้คือเกมทายตัวเลขครับ พี่จะกำหนดเลขความลับไว้ 1 ตัว สมมติว่าเป็นเลข 42 น้องๆ ต้องทายมาเรื่อยๆ ถ้าทายมากไป พี่จะบอกว่า 'Too high!' ถ้าทายน้อยไป พี่จะบอก 'Too low!' จนกว่าจะทายถูกเป๊ะๆ"
"คำถามคือ เราควรใช้ Loop ชนิดไหนดี ระหว่าง for กับ while? ... คำตอบคือ **while loop** ครับ! เพราะ for loop เราใช้เมื่อ 'รู้จำนวนรอบที่แน่นอน' แต่เกมนี้เราไม่รู้เลยว่าผู้เล่นจะทายกี่รอบถึงจะถูก ดังนั้น while (ทำตราบเท่าที่เงื่อนไขเป็นจริง) จึงเหมาะสมที่สุดครับ"

"**Algorithm:**"
"1. ตั้งค่าเลขความลับ `secret = 42`"
"2. สร้างตัวแปร `guess` มารับค่าที่ผู้เล่นทาย"
"3. เราจะใช้ลูปเงื่อนไขว่า `while (guess != secret)` คือ 'วนทำไปเรื่อยๆ ตราบเท่าที่ยังทายไม่ถูก'"
"4. ในลูป: รับค่าทายจากผู้เล่น → เช็คว่า > หรือ < เลขความลับ เพื่อใบ้คำ"
"5. ถ้าทายถูกเมื่อไหร่ มันจะหลุดออกจากลูป เราค่อยปริ้นต์บอกว่า 'ทายถูกแล้ว!'"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โจทย์:** ทายตัวเลขความลับ (42) มีคำใบ้ "มากไป/น้อยไป"
- **Algorithm:** กำหนดค่าเริ่มต้น → while (guess != secret) → รับค่า guess → if-else เช็คการใบ้ → หลุดลูปพิมพ์ "ถูกต้อง!"
- **เปรียบเทียบ:** for (รู้รอบแน่นอน) vs while (ไม่รู้รอบ)

**💡 เทคนิคการสอน:**
- ชวนเด็กเล่นเกมนี้กันจริงๆ ก่อน โดยพี่สอนคิดเลขในใจ แล้วให้น้องๆ ในห้องสุ่มทายเลข เพื่อให้เข้าใจลอจิกการใบ้คำ

### Slide 3.2: โค้ดและการตั้งค่าเริ่มต้น (Initialization)
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"มาดูโค้ดกันครับ มีจุดสำคัญมากๆ ที่ทุกคนห้ามพลาด!"

```c
#include <stdio.h>

int main() {
    int secret = 42;
    int guess = -1;  // จุดสำคัญ: ค่าเริ่มต้นต้องไม่เท่ากับ secret!

    printf("=== Guess the Number! ===\n");

    while (guess != secret) {
        printf("Enter your guess: ");
        scanf("%d", &guess);

        if (guess > secret) {
            printf("Too high!\n");
        } else if (guess < secret) {
            printf("Too low!\n");
        }
    }

    printf("Correct! The answer is %d!\n", secret);
    return 0;
}
```

"ข้อควรระวังขั้นสุดยอด! ตอนเราประกาศตัวแปร `guess` พี่ตั้งค่ามันเป็น `-1` (หรืออะไรก็ได้ที่ไม่ใช่ 42) ทำไมต้องทำแบบนั้น?"
"ลองนึกภาพว่าถ้าพี่เผลอตั้งค่า `int guess = 42;` พอโปรแกรมมาถึงบรรทัด `while (guess != secret)` มันจะแปลว่า `while (42 != 42)` ซึ่งเป็น **เท็จ** ตั้งแต่แรก! ผลคือโปรแกรมจะไม่ยอมเข้าไปทำงานในลูปเลย เกมจบตั้งแต่ยังไม่เริ่มครับ!"
"ดังนั้น เราต้องหลอกโปรแกรมด้วยการให้ค่าเริ่มต้นที่ไม่ตรงกับความจริง เพื่อบังคับให้มันเข้าลูปครั้งแรกให้ได้นั่นเอง"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โค้ดเต็ม + Output ตัวอย่าง** (แสดง History การทายผิดจนถึงทายถูก)
- **Key Insight ตัวใหญ่ๆ:** `int guess = -1;` ต้องตั้งค่าเริ่มต้นที่ไม่ตรงกับ `secret` เพื่อบังคับเข้า while loop

**💡 เทคนิคการสอน:**
- ให้น้องที่ทำเสร็จแล้ว ลองแก้บรรทัด `int guess = 42;` แล้วรันดู จะได้เข้าใจว่าทำไมมันไม่ทำงาน
- **โจทย์ท้าทาย (Bonus):** "ใครทำเสร็จแล้ว ลองเพิ่มตัวแปรเพื่อนับจำนวนครั้งที่ทายดูสิ แล้วตอนจบให้โชว์ว่า 'You got it in X tries!'"

## 🏆 Workshop 4: พีระมิดดวงดาว (Star Pyramid)

### Slide 4.1: วิเคราะห์รูปแบบลูปซ้อนลูป (Nested Loop)
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"ข้อนี้พี่บอกเลยว่า เป็นโจทย์คลาสสิกที่โปรแกรมเมอร์ทุกคนต้องเคยผ่านมาครับ! เราจะมาวาดรูปด้วยสัญลักษณ์ `*` กัน"
"ข้อนี้ต้องใช้เทคนิคที่เรียกว่า **Nested Loop** หรือ **ลูปซ้อนลูป** ครับ ฟังดูเหมือนจะยาก แต่มันมีรูปแบบของมันอยู่"

"สมมติพี่ป้อน N = 4 พี่อยากได้รูปนี้ครับ:"
```
*
**
***
****
```
"ก่อนจะเขียนโค้ด เรามาหาแพทเทิร์นกันก่อน สังเกตอะไรไหมครับ?"

"- บรรทัดที่ 3: เราพิมพ์ดาว 3 ดวง"

"เห็นไหมครับว่า **จำนวนดวงดาวในแต่ละบรรทัด มันจะเท่ากับ หมายเลขของบรรทัดนั้นเป๊ะเลย!**"
"เวลาเราวาดรูปด้วยโปรแกรม เราต้องใช้ลูป 2 ตัวทำงานร่วมกันครับ:"
"- **ลูปนอก (สมมติให้ชื่อตัวแปร i):** จะทำหน้าที่ควบคุม **จำนวนบรรทัด** ว่าจะลงมาทำกี่บรรทัด (จาก 1 ถึง N)"
"- **ลูปใน (สมมติให้ชื่อตัวแปร j):** จะทำหน้าที่ควบคุม **การพิมพ์ดวงดาวในแนวนอนของแต่ละบรรทัด** (จาก 1 ถึง i)"

"คิดง่ายๆ ครับ ลูปนอกเหมือนสั่งหัวหน้าว่า 'วันนี้ให้ทำงาน N วันนะ' แล้วหัวหน้าก็ไปสั่งลูกน้อง (ลูปใน) ว่า 'ในวันที่ i ให้เธอพิมพ์ดาว i ดวงนะ' พอพิมพ์เสร็จ ก็ค่อยขึ้นบรรทัดใหม่"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **รูป Output:** สามเหลี่ยมดาว N ชั้น
- **Algorithm & Pattern Matching:** โชว์ความสัมพันธ์ว่า จำนวนดาว = ลำดับบรรทัด
- **โครงสร้าง Nested Loop:** ลูปนอกจัดการบรรทัด (i) → ลูปในจัดการดาว (j) → จบแต่ละบรรทัดด้วย `\n`

**💡 เทคนิคการสอน:**
- วาดตารางบนกระดานให้เด็กดูความสัมพันธ์ของ i (บรรทัด) และ j (ดวงดาว) ช้าๆ เพื่อให้เห็นภาพว่ามันทำงานอย่างไร

### Slide 4.2: การเขียนโค้ดและ Trace การทำงาน
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"มาดูหน้าตาของโค้ดกันครับ"

```c
#include <stdio.h>

int main() {
    int n;
    printf("Enter N: ");
    scanf("%d", &n);

    // ลูปนอก: จัดการทีละบรรทัด
    for (int i = 1; i <= n; i++) {       
        // ลูปใน: พิมพ์ดาวตามจำนวน i
        for (int j = 1; j <= i; j++) {   
            printf("*");
        }
        printf("\n");  // พอพิมพ์ดาวเสร็จในแต่ละบรรทัด ต้องขึ้นบรรทัดใหม่
    }
    return 0;
}
```

"จุดที่เป็นหัวใจสำคัญของข้อนี้คือขีดจำกัดของลูปในครับ สังเกตตรง `j <= i` นะครับ มันไม่ได้เป็น `j <= n`"
"มา Trace (ไล่สเต็ป) การทำงานกันดู:"
"- รอบแรก **i = 1**: ลูปในเริ่มจาก j = 1 ถึง 1 → วนรอบเดียว พิมพ์ `*` 1 ดวง → หลุดลูปในไปเจอ `\n` ขึ้นบรรทัดใหม่"
"- รอบสอง **i = 2**: ลูปในเริ่มจาก j = 1 ถึง 2 → วนสองรอบ พิมพ์ `**` → เจอ `\n` ขึ้นบรรทัดใหม่"
"- รอบสาม **i = 3**: ลูปในเริ่มจาก j = 1 ถึง 3 → วนสามรอบ พิมพ์ `***` → เจอ `\n` ขึ้นบรรทัดใหม่"
"เห็นไหมครับ ลูปในขยันพิมพ์ตามคำสั่งของ i เสมอ! ถ้าเราเข้าใจลอจิกนี้ เราจะวาดรูปอะไรก็ได้บนคอนโซลครับ"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โค้ดเต็ม**
- **Highlight:** เงื่อนไขของลูปใน `j <= i` (ไม่ใช่ n) และตำแหน่งของ `printf("\n");` ที่ต้องอยู่ *นอก* ลูปใน แต่ *ใน* ลูปนอก
- **Trace Table (Animation):** แสดงการทำงานไปทีละบรรทัด (i) ทีละคอลัมน์ (j)

**💡 เทคนิคการสอน:**
- **โจทย์ท้าทายระดับเซียน:** "ใครเสร็จแล้ว ลองพิมพ์สามเหลี่ยม **กลับหัว** ดูสิ! (บรรทัดแรกมี N ดวง บรรทัดสุดท้ายมี 1 ดวง)" (ใบ้ให้: เปลี่ยนให้ลูปนอกนับถอยหลัง)

"พี่ให้ลองจินตนาการว่า น้องเป็นกรรมการประกวด Miss Universe นะครับ มีสาวงามเดินออกมาทีละคน น้องต้องหาคนที่สวยที่สุด น้องจะทำยังไง?"

### Slide 5.2: Infinite Loop และ Break
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"มาดูโค้ดกันครับ ข้อนี้เราจะใช้เทคนิคใหม่ที่เรียกว่า **Infinite Loop** หรือ ลูปอนันต์ คู่กับคำสั่ง **break**"

```c
#include <stdio.h>

int main() {
    int score;
    int max_score = 0;  // ตั้งค่าแชมป์เริ่มต้นให้ต่ำๆ ไว้ก่อน

    printf("Enter scores (type -1 to stop):\n");

    // while(1) คือลูปที่ไม่มีวันจบ (เพราะ 1 แปลว่า True เสมอ)
    while (1) {  
        printf("Score: ");
        scanf("%d", &score);

        // เช็คทางออกฉุกเฉิน
        if (score == -1) {
            break;  // คำสั่งทะลวงออกจากลูปทันที!
        }

        // เช็คการเปลี่ยนแชมป์
        if (score > max_score) {
            max_score = score;  // สถาปนาแชมป์คนใหม่!
        }
    }

    printf("Maximum score: %d\n", max_score);
    return 0;
}
```

"เราใช้ `while (1)` ซึ่งแปลว่า 'ทำไปเรื่อยๆ ไม่มีวันหยุด' แต่เรามี **ประตูทางออกฉุกเฉิน** อยู่ข้างในลูป นั่นคือคำสั่ง `break;` ครับ เมื่อผู้ใช้ป้อน `-1` เมื่อไหร่ `break` จะกระโดดถีบเราออกจาก `while` ลูปทันที!"
"ถ้าเราลองไล่เช็ค (Trace) สมมติป้อน `70, 85, 60, 90, -1`:
- ป้อน 70: 70 > 0 ไหม? จริง! ตอนนี้ `max_score` กลายเป็น 70
- ป้อน 85: 85 > 70 ไหม? จริง! `max_score` โดนโค่น กลายเป็น 85
- ป้อน 60: 60 > 85 ไหม? ไม่จริง! `max_score` รอดตัว ยังคงเป็น 85
- ป้อน 90: 90 > 85 ไหม? จริง! ได้แชมป์ใหม่ `max_score` กลายเป็น 90
- ป้อน -1: เจอ `break;` ออกจากลูป!"
"ในที่สุด เราก็จะได้คะแนน 90 ไปแสดงผลครับ"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โค้ดเต็ม + Output ตัวอย่าง**
- **Highlight:** `while(1)` คู่กับคำสั่ง `break;` (ทางออกฉุกเฉิน)
- **Trace Table:** โชว์การเปลี่ยนค่าของ `max_score` ทุกครั้งที่รับค่าใหม่เข้ามา

**💡 เทคนิคการสอน:**
- **โจทย์ท้าทาย (Bonus):** "ถ้าพี่อยากหา **ค่าต่ำสุด (Min)** ด้วยล่ะ? เราต้องเปลี่ยนเครื่องหมายในเงื่อนไขยังไง? และควรตั้งค่า `min_score` เริ่มต้นที่เท่าไหร่ดี?" (ใบ้: ต้องตั้งให้สูงลิบลิ่วไว้ก่อน)

**💡 เทคนิคการสอน:**

### Slide 6.2: โค้ดและการคำนวณ VAT
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"เรามาดูโค้ดจริงกันครับ โค้ดจะถูกแบ่งเป็น 2 ส่วน คือส่วนของฟังก์ชัน และส่วนของ `main`"

```c
#include <stdio.h>

// ฟังก์ชันสำหรับคำนวณยอดรวมและบวก VAT
float calc_total(int arr[], int size) {
    int sum = 0;
    for (int i = 0; i < size; i++) {
        sum = sum + arr[i];  // สะสมค่าไปเรื่อยๆ
    }
    return sum * 1.07;  // เอาผลรวมคูณ 1.07 คือการบวก VAT 7%
}

int main() {
    int prices[5];

    // วนลูปรับค่าราคาสินค้าทีละชิ้น
    for (int i = 0; i < 5; i++) {
        printf("Enter price %d: ", i + 1);
        scanf("%d", &prices[i]);
    }

    // เรียกใช้งานฟังก์ชัน โดยส่งชื่อ array และขนาด(5) เข้าไป
    float total = calc_total(prices, 5);
    
    // พิมพ์ผลลัพธ์
    printf("Total with VAT 7%%: %.2f baht\n", total);
    return 0;
}
```

"พี่ขอชี้เป้า 2 จุดสำคัญครับ:
จุดที่ 1: ตอนเราเรียกใช้ฟังก์ชัน เราเขียนแค่ `calc_total(prices, 5);` สังเกตไหมว่าตอนส่งไป เราพิมพ์แค่ชื่อ Array `prices` เฉยๆ ไม่ต้องมี `[]` หรือ `&` เลย!
จุดที่ 2: ดูตรงบรรทัดคำสั่ง `printf` ครับ มีใครเห็นสัญลักษณ์ประหลาดไหม? `%%` มันคืออะไร?
คำตอบคือ ถ้าเราอยากพิมพ์ตัวอักษร `%` (เปอร์เซ็นต์) โชว์ออกไปบนหน้าจอจริงๆ เราต้องเขียนเบิ้ล 2 ตัวเป็น `%%` ครับ! เพราะถ้าเขียนตัวเดียว ภาษา C จะนึกว่าเรากำลังจะใช้ Format Specifier (เช่น `%d`, `%f`) นั่นเอง"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โค้ดเต็ม + Output ตัวอย่าง**
- **Highlight:** วิธีการเรียกฟังก์ชัน `calc_total(prices, 5)`
- **Tip น่ารู้:** การพิมพ์สัญลักษณ์ `%` บนหน้าจอ ต้องใช้ `%%` ใน `printf`

**💡 เทคนิคการสอน:**
- ให้เด็กๆ ลองแก้ไขโค้ด จาก VAT 7% เปลี่ยนเป็น VAT 10% ดูว่าพวกเขาจะปรับตัวเลขเป็นอะไร (คูณ 1.10)
- ถามย้ำ: "ทำไมตอนโยนค่าเข้า `calc_total` เราถึงโยนแค่ `prices` ไม่ใส่ `&`?" (เพราะชื่อ Array เปรียบเสมือน Address อยู่แล้ว)

## 🏆 Workshop 7: ค้นหานักเรียนที่สอบผ่าน (Pass/Fail Finder)

### Slide 7.2: โค้ดและการใช้ Void Function
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"มาดูโค้ดกันครับ ข้อนี้เราจะใช้ฟังก์ชันประเภท `void`"

```c
#include <stdio.h>

// ฟังก์ชันประเภท void จะไม่มีการ return ค่ากลับ
void count_results(int scores[], int size) {
    int pass = 0, fail = 0;  // ตัวแปรนับจำนวน ต้องเริ่มต้นที่ 0 เสมอ!

    for (int i = 0; i < size; i++) {
        if (scores[i] >= 50) {
            pass++;  // เจอคนผ่าน บวกแต้มเข้าไป
        } else {
            fail++;  // เจอคนตก บวกแต้มเข้าไป
        }
    }

    // พิมพ์ผลสรุปข้างในฟังก์ชันนี้เลย
    printf("Passed: %d students\n", pass);
    printf("Failed: %d students\n", fail);
}

int main() {
    // ประกาศและกำหนดค่าเริ่มต้นให้ Array 10 ช่องรวดเดียวเลย
    int scores[10] = {75, 42, 88, 35, 90, 55, 48, 67, 30, 82};
    
    // ส่งไปให้ฟังก์ชันจัดการ
    count_results(scores, 10);
    
    return 0;
}
```

"น้องๆ เห็นไหมครับว่า ฟังก์ชัน `count_results` ถูกนำหน้าด้วยคำว่า `void`
`void` แปลว่า 'ความว่างเปล่า' หมายความว่าฟังก์ชันนี้จะทำงานของมันไปเรื่อยๆ พิมพ์ผลลัพธ์ออกหน้าจอด้วยตัวเอง แล้วก็จบไปเลย ไม่ต้องมีการเตะส่งค่า (return) กลับมาให้ `main` เหมือนข้อที่แล้วๆ มา
เวลาเรียกใช้ใน `main` เราก็แค่เรียกชื่อฟังก์ชันเฉยๆ ไม่ต้องหาตัวแปรมารับค่าอะไรเลย สะดวกมากๆ ครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โค้ดเต็ม + Output ตัวอย่าง**
- **Highlight 1:** การตั้งค่าเริ่มต้นตัวนับ `int pass = 0, fail = 0;` ต้องเซ็ตเป็น 0 เสมอ
- **Highlight 2:** อธิบายว่า `void` function คืออะไร (ไม่ต้องมี `return`)

**💡 เทคนิคการสอน:**
- ให้เด็กลองเปลี่ยนเกณฑ์คะแนนผ่านจาก 50 เป็น 60 ดูซิว่ามีคนรอดกี่คน
- **โจทย์ท้าทาย (Bonus):** "ไหนใครสามารถเพิ่มตัวแปรเพื่อนับคนได้เกรด A (คะแนนตั้งแต่ 80 ขึ้นไป) เข้าไปในฟังก์ชันนี้ได้บ้าง?"

## 🏆 Workshop 8: สลับตัวอักษร (Word Reverser)

### Slide 8.1: ลอจิกการทำงานย้อนกลับ (Reverse Thinking)
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"ข้อที่ 8 ครับ ข้อนี้เจ๋งมาก เราจะมาเรียนรู้การสลับตัวอักษร หรือทำสิ่งที่เราเรียกว่า **Reverse Thinking (กระบวนการคิดสวนทาง)** กัน!"
"โจทย์คือ ถ้ารับคำว่า 'hello' เข้ามา เราอยากให้โปรแกรมพ่นคำว่า 'olleh' ออกมาแทน (อ่านจากหลังมาหน้า)"

"ลอจิกในการทำแบบนี้คืออะไร?"
"1. เรารับคำเข้ามาเก็บไว้ใน Array ของอักขระ (ก็คือ String นั่นเอง)"
"2. เราต้องรู้ก่อนว่าคำๆ นั้นมีความยาวกี่ตัวอักษร โดยใช้ฟังก์ชัน `strlen()` (String Length) ซึ่งเราต้องอัญเชิญ Library ที่ชื่อ `<string.h>` มาใช้ครับ"
"3. พระเอกของเราคือ `for` loop ครับ! ปกติเราจะให้ loop เริ่มนับจาก 0 ไปข้างหน้าใช่ไหม? แต่คราวนี้ เราจะให้มันเริ่มจาก **ท้ายสุด แล้วนับถอยหลัง** ลดค่าไปจนถึงช่องที่ 0 ครับ"
"4. ในลูป เราก็แค่สั่งปริ้นต์ตัวอักษรออกมาทีละตัว"

"มีข้อควรระวังนิดนึงครับ... สมมติว่าความยาวของคำคือ len เราต้องให้ลูปเริ่มจากช่องที่ `len - 1` นะครับ ไม่ใช่เริ่มจาก `len`!
ทำไมล่ะ? ใครจำได้บ้าง... ถูกต้องครับ! เพราะ Array ในภาษา C เริ่มนับช่องจาก Index 0!
ดังนั้น คำว่า 'hello' มี 5 ตัวอักษร มันจะอยู่ในช่อง index 0, 1, 2, 3, และ 4... ตัวสุดท้ายจึงอยู่ที่ index 4 (ซึ่งก็คือ 5 - 1 นั่นเอง) จำไว้ให้แม่นเลยนะครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โจทย์:** รับ 1 คำ → พิมพ์ตัวอักษรย้อนกลับ (hello → olleh)
- **ภาพประกอบ Array:** วาดรูปกล่อง Array บรรจุตัว h, e, l, l, o พร้อมกำกับเลข Index 0, 1, 2, 3, 4 ชี้ให้เห็นว่าช่องสุดท้ายคือ len - 1
- **Algorithm Flowchart:** หาความยาวด้วย strlen → for (i = len-1; i >= 0; i--) → printf ทีละตัว

**💡 เทคนิคการสอน:**
- ยกมือขึ้นมานับนิ้วโชว์เลยครับ "นิ้วโป้งเบอร์ 0, นิ้วชี้เบอร์ 1... มี 5 นิ้ว นิ้วก้อยคือเบอร์ 4!" ให้เด็กเห็นภาพชัดๆ

### Slide 8.2: โค้ดและการใช้ String
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"ทีนี้เรามาดูโค้ดกันครับ มีเซอร์ไพรส์รออยู่ตรงบรรทัดที่รับค่าด้วย!"

```c
#include <stdio.h>
#include <string.h>  // อย่าลืม include ตัวนี้ เพื่อใช้คำสั่ง strlen

int main() {
    char word[100];
    
    printf("Enter a word: ");
    scanf("%s", word);  // เซอร์ไพรส์! ไม่ต้องใส่ & หน้าตัวแปร String

    int len = strlen(word);  // หาความยาวของคำ

    printf("Reversed: ");
    
    // ลูปพระเอก: วนนับถอยหลัง
    for (int i = len - 1; i >= 0; i--) {
        printf("%c", word[i]);  // พิมพ์ทีละตัว ย้อนกลับ
    }
    printf("\n");
    
    return 0;
}
```

"เห็นบรรทัด `scanf` ไหมครับพี่น้อง! `scanf("%s", word);` **ไม่มีเครื่องหมาย `&` นำหน้าชื่อตัวแปร!**
เกิดอะไรขึ้น? ปกติสอนมาตลอดว่า scanf ต้องมี `&` เสมอนี่นา?
นี่แหละครับ คือ **ข้อยกเว้นเพียงหนึ่งเดียว** ในภาษา C! เพราะตัวแปรประเภท Array (และ String ก็คือ Array ของ char) ตัวชื่อของมันเอง (คำว่า `word`) ทำหน้าที่บอก Address หรือที่อยู่ของตัวมันเองอยู่แล้ว เราเลยไม่ต้องเอา `&` ไปแปะหน้ามันอีกครับ พิเศษจริงๆ!"

"มาลอง Trace ลูปดูครับ ถ้าคำว่า 'hello' ความยาวคือ 5 (len = 5)
- i = 4: ดึง `word[4]` มาพิมพ์ จะได้ตัว 'o'
- i = 3: ดึง `word[3]` มาพิมพ์ จะได้ตัว 'l'
- i = 2: ดึง `word[2]` มาพิมพ์ จะได้ตัว 'l'
- i = 1: ดึง `word[1]` มาพิมพ์ จะได้ตัว 'e'
- i = 0: ดึง `word[0]` มาพิมพ์ จะได้ตัว 'h'
- พอ i ติดลบ ก็หลุดลูป จบปิ๊ง! พิมพ์ออกมาเป็น 'olleh' พอดีเป๊ะ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โค้ดเต็ม + Output ตัวอย่าง**
- **Highlight ใหญ่โต:** รับ String ด้วย `%s` **ห้าม** ใส่ `&` นำหน้า!
- **Library ย้ำเตือน:** ต้องใส่ `#include <string.h>` ด้วยนะ ไม่งั้นใช้ `strlen()` ไม่ได้
- **Trace Table การนับถอยหลังของลูป**

**💡 เทคนิคการสอน:**
- แกล้งเขียนโค้ดใส่ `&word` แล้วรันให้เด็กดูว่าคอมไพเลอร์บางตัวจะร้องเตือน (Warning)
- **โจทย์ท้าทาย (Bonus):** "ใครรู้บ้างว่า คำว่า Palindrome คืออะไร? มันคือคำที่อ่านจากหน้าไปหลัง หรือหลังไปหน้า ก็อ่านเหมือนกัน เช่น 'radar' พี่ขอท้าให้เขียนโปรแกรมเช็คว่าคำที่ป้อนมาเป็น Palindrome หรือเปล่า!"

## 🏆 Workshop 9: ตู้เอทีเอ็มจำลอง (Personal ATM)

### Slide 9.1: การบริหารจัดการสถานะ (State Management)
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"มาถึงโค้งสุดท้ายแล้ว! ข้อที่ 9 นี้ น้องๆ จะได้ทำโปรเจกต์ที่จำลองระบบชีวิตจริงมากๆ นั่นคือ **ตู้ ATM ส่วนตัว!**"
"ATM ของเราจะมีความสามารถ 3 อย่างครับ:
1. ฝากเงิน (Deposit)
2. ถอนเงิน (Withdraw)
3. เช็คยอดเงินคงเหลือ (Check Balance)"
"และแน่นอน มันต้องมีเมนูที่ 4 คือ 'ออกจากระบบ (Exit)' ด้วย"

"ลอจิกหลักของข้อนี้คือสิ่งที่เราเรียกว่า **การบริหารจัดการสถานะ หรือ State Management** ครับ
ยอดเงิน (Balance) ของเรา ถือเป็น 'สถานะหลัก' ของโปรแกรม สมมติเรามีเงินตั้งต้น 1,000 บาท
- ถ้าเราเลือกฝาก 500 บาท → สถานะเงินต้องกลายเป็น 1,500 บาท
- ถ้าเราไปถอน 200 บาท → สถานะเงินต้องหักจาก 1,500 เหลือ 1,300 บาท
เห็นไหมครับว่า ทุกครั้งที่เราทำธุรกรรม ค่า balance จะถูกอัปเดตและเปลี่ยนแปลงไปเรื่อยๆ นี่แหละคือหัวใจหลัก!"

"เราจะสร้างโปรแกรมโดยใช้เมนูลูปอนันต์ `while(1)` เพื่อให้ตู้ ATM ทำงานวนไปเรื่อยๆ จนกว่าลูกค้าจะกดเลือกเมนู Exit เราถึงจะเตะเขาออกด้วยคำสั่ง `break` ครับ"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **System Design Overview:** แสดงโครงสร้างระบบว่าประกอบด้วย Main Loop และแบ่งออกเป็น 3 Functions (Deposit, Withdraw, Check Balance)
- **Concept "State Management":** วาดภาพตัวแปร Balance เป็นกล่องสมบัติ ที่ถูกแก้ไขค่าตลอดเวลาด้วย Function ต่างๆ
- **Flowchart:** เมนูหลัก → เลือกช้อยส์ → เข้าฟังก์ชัน → นำผลลัพธ์กลับมาอัปเดตกล่องสมบัติ

**💡 เทคนิคการสอน:**
- เปรียบเทียบตัวแปร Balance เป็นสมุดบัญชีเงินฝากที่ต้องเอาไปอัปเดตทุกครั้งหลังทำธุรกรรม

### Slide 9.2: การเขียนโค้ดผสาน Function เข้าด้วยกัน
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"โปรแกรมนี้จะดูยาวนิดนึง แต่มันเกิดจากการเอาชิ้นส่วนเล็กๆ (Functions) มาต่อกันแบบเลโก้ครับ"

```c
#include <stdio.h>

// ฟังก์ชันฝากเงิน (คืนค่ายอดเงินใหม่กลับไป)
float deposit(float balance, float amount) {
    return balance + amount;
}

// ฟังก์ชันถอนเงิน พร้อมระบบป้องกันถอนเกิน
float withdraw(float balance, float amount) {
    if (amount > balance) {
        printf("Not enough balance!\n");
        return balance;  // เงินไม่พอ คืนยอดเดิมไปเลย ไม่หักเงิน
    }
    return balance - amount; // หักเงินแล้วคืนยอดใหม่
}

// ฟังก์ชันเช็คยอดเงิน (เป็นแค่หน้าปัดแสดงผล เลยใช้ void)
void check_balance(float balance) {
    printf("Current balance: %.2f baht\n", balance);
}

int main() {
    float balance = 1000.0;  // สถานะเริ่มต้น!
    int choice;
    float amount;

    while (1) {  // ลูปเมนู ATM ตลอดกาล
        printf("\n=== Personal ATM ===\n");
        printf("1. Deposit\n");
        printf("2. Withdraw\n");
        printf("3. Check Balance\n");
        printf("4. Exit\n");
        printf("Choose: ");
        scanf("%d", &choice);

        if (choice == 1) {
            printf("Enter amount to deposit: ");
            scanf("%f", &amount);
            // หัวใจหลัก: เอาเงินเก่า โยนเข้า deposit ได้เงินใหม่ เอามาทับยอดเก่า
            balance = deposit(balance, amount);
            printf("Deposited %.2f baht\n", amount);
        } else if (choice == 2) {
            printf("Enter amount to withdraw: ");
            scanf("%f", &amount);
            // เหมือนกัน: โยนเงินเก่าเข้า withdraw แล้วอัปเดตยอดกลับ
            balance = withdraw(balance, amount);
        } else if (choice == 3) {
            check_balance(balance);
        } else if (choice == 4) {
            printf("Goodbye!\n");
            break;  // จบการทำงาน
        } else {
            printf("Invalid choice!\n");
        }
    }
    return 0;
}
```

"จุดที่เจ๋งที่สุดคือในฟังก์ชัน `withdraw` ครับ!
พี่แอบดัก Error ไว้ด้วย... ถ้าเกิดคนพยายามถอนเงิน 'มากกว่า' ยอดเงินที่มี ฟังก์ชันจะพิมพ์ด่าว่า 'Not enough balance!' จากนั้นมันจะ **return ยอดเงินเดิมกลับไปหน้าตาเฉย** ครับ (หมายความว่าไม่ยอมหักเงินให้นั่นเอง) ระบบ ATM ของเราก็จะปลอดภัย ไม่มีใครถอนเงินจนบัญชีติดลบได้เด็ดขาด!"
"และสังเกตในเมนูข้อ 1 และ 2 นะครับ เราใช้คำสั่ง `balance = deposit(...)` นั่นคือเราต้องนำผลลัพธ์ที่ return กลับมา เซฟทับลงในตัวแปร `balance` เสมอ เพื่ออัปเดตสมุดบัญชีของเราครับ"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โค้ดเต็ม + Output การจำลองกด ATM สลับไปมา**
- **Highlight:** การอัปเดตสถานะด้วย `balance = function(balance, amount);`
- **Highlight:** ระบบป้องกันถอนเงินติดลบในฟังก์ชัน `withdraw`

**💡 เทคนิคการสอน:**
- ชวนเด็กๆ ในห้องมาเล่นบทบาทสมมติเป็นลูกค้ากด ATM จริงๆ ฝาก ถอน เช็คยอด สลับไปมาให้ดูว่าเงินมันเปลี่ยนตามที่บอกไหม

## 🏆 Workshop 10: ระบบสุ่มผู้โชคดี (Lucky Draw)

### Slide 10.1: ความลับของการสุ่ม (Pseudo-Random)
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"เดินทางมาถึงข้อสุดท้ายแล้วครับทุกคน! ข้อนี้จะเป็นการนำศาสตร์มืด... เอ้ย! นำ Library ภายนอกมาใช้งานครับ เราจะสร้างระบบวงล้อสุ่มผู้โชคดี (Lucky Draw) กัน!"
"น้องๆ รู้ไหมครับว่า ปกติคอมพิวเตอร์เป็นเครื่องจักรที่ทำงานตรงไปตรงมาเป๊ะๆ สั่ง 1 ได้ 1 สั่ง 2 ได้ 2... ดังนั้นโดยพื้นฐานแล้ว คอมพิวเตอร์ **สุ่มเลขไม่เป็นครับ!!**"
"อ้าว แล้วในเกมกาชาปอง เขาสุ่มเกลือให้เราได้ยังไงล่ะ? คำตอบคือ เราต้องใช้การคำนวณทางคณิตศาสตร์ที่ซับซ้อนมากๆ สร้างตัวเลขที่ดูเหมือนจะเดาทางไม่ได้ขึ้นมา เราเรียกมันว่า **Pseudo-random number (ตัวเลขสุ่มหลอก)**"

"และในภาษา C เราจะใช้อาวุธจาก Library สองตัวมาช่วยครับ คือ `<stdlib.h>` และ `<time.h>`"
"กระบวนการสุ่มมี 3 ขั้นตอนจำง่ายๆ ครับ:
1. **ตั้งฐานการสุ่ม (Set Seed):** ด้วยคำสั่ง `srand(time(NULL));` เราจะใช้ 'เวลาปัจจุบัน' เป็นฐานในการคำนวณสูตรสุ่ม เพราะเวลาบนโลกเปลี่ยนไปทุกวินาที ทำให้ค่าที่สุ่มออกมาได้ ไม่ซ้ำของเดิม! (คำสั่งนี้ทำแค่ครั้งเดียวพอ ตอนเริ่ม main)
2. **กดปุ่มสุ่ม:** ด้วยคำสั่ง `rand()` มันจะคายตัวเลขมหาศาลออกมา เช่น 1,842,913 
3. **ตีกรอบตัวเลข:** เนื่องจากเลขมันใหญ่ไป เราจะใช้เครื่องหมาย `%` (Modulus ที่เรารัก) มาตีกรอบมัน เช่น ถ้าเราอยากได้เลข 0 ถึง 4 เราก็เอาไป `% 5` ครับ! (จำได้ไหมว่าหารเอาเศษด้วย 5 เลขที่เป็นไปได้คือ 0, 1, 2, 3, และ 4 เท่านั้น)"

"เป้าหมายของเราคือ เอาเลข 0-4 ที่สุ่มได้นี้ ไปใช้เป็น Index (ตำแหน่ง) ชี้เข้าไปใน Array เพื่อล้วงเอาชื่อคนโชคดีออกมาครับ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โจทย์:** สุ่มชื่อผู้โชคดี 1 คน จากใน Array ที่เตรียมไว้ 5 ชื่อ
- **Concept: คอมพิวเตอร์สุ่มเลขไม่เป็น!** แนะนำคำว่า Pseudo-random number
- **3 ขั้นตอนการสุ่ม (สไลด์ไฮไลท์):**
  1. Set Seed: `srand(time(NULL));` (ใส่ครั้งเดียว)
  2. Roll Dice: `rand()` (ได้เลขใหญ่ยักษ์)
  3. Modulus Limit: `% 5` (บีบให้อยู่ในกรอบ 0-4)

**💡 เทคนิคการสอน:**
- อธิบายเรื่อง Seed ว่า ถ้าไม่ใส่ srand() คอมไพเลอร์จะใช้เลข Seed เดิมเสมอ ทำให้ทุกครั้งที่ปิดและเปิดโปรแกรมใหม่ ลำดับเลขสุ่มจะเหมือนเดิมเป๊ะๆ (เรียกว่า เดาทางได้)

### Slide 10.2: โค้ด 2D Array และการแสดงผล
**🎤 บทพูด (สิ่งที่ต้องพูด):**
"ลุยเขียนโค้ดเลยครับ! ข้อนี้พี่จะแถมความรู้นิดนึงเรื่อง Array 2 มิติด้วย"

```c
#include <stdio.h>
#include <stdlib.h>  // อัญเชิญ library สำหรับ rand() และ srand()
#include <time.h>    // อัญเชิญ library สำหรับ time()

int main() {
    // Array เก็บ String หลายๆ ตัว (เรียกว่า Array 2 มิติ)
    // 5 คือจำนวนรายชื่อ, 20 คือความยาวสูงสุดของแต่ละชื่อ
    char names[5][20] = {"Alice", "Bob", "Charlie", "Diana", "Elon"};

    // 1. ตั้ง Seed โดยใช้เวลาปัจจุบัน (ย้ำ: ทำแค่บรรทัดนี้บรรทัดเดียว ครั้งเดียว!)
    srand(time(NULL));  

    // 2 & 3. สุ่มเลขใหญ่ๆ แล้ว Mod ด้วย 5 เพื่อให้เหลือ 0, 1, 2, 3, หรือ 4
    int random_index = rand() % 5;  

    printf("=== Lucky Draw! ===\n");
    // เอาตัวเลขสุ่มที่ได้ ไปเป็น Index เพื่อล้วงเอาชื่อออกมาแสดง
    printf("The winner is: %s!\n", names[random_index]);
    
    return 0;
}
```

"สังเกตตรงบรรทัดที่เก็บชื่อนะครับ พี่ประกาศว่า `char names[5][20]` นี่คือ **Array 2 มิติ** ครับ!
มิติแรก (เลข 5) บอกว่า เรามี 5 แถว หรือ 5 คน
มิติที่สอง (เลข 20) บอกว่า ในแต่ละคน อนุญาตให้พิมพ์ชื่อได้ยาวไม่เกิน 20 ตัวอักษร
น้องๆ ยังไม่ต้องเครียดเรื่อง 2 มิติมากนะครับ ให้รู้ว่ามันคือการเก็บกลุ่มของคำ (Array of Strings) ก็พอ เวลาเรียกใช้งาน เราก็แค่ระบุมิติแรก `names[random_index]` มันก็จะปริ้นต์ชื่อนั้นออกมาเลย!"

"เมื่อน้องรันโปรแกรม น้องจะได้ผู้ชนะต่างกันไปในแต่ละครั้ง เช่น รันรอบแรกอาจได้ Charlie ปิดแล้วรันใหม่รอบสองอาจได้ Alice นี่แหละครับระบบสุ่มที่สมบูรณ์แบบ!"

**📊 สิ่งที่ต้องแสดงในสไลด์:**
- **โค้ดเต็ม + Output (โชว์ผลการรัน 2-3 ครั้งที่ผลลัพธ์ไม่เหมือนกัน)**
- **Highlight:** Include ตัว `<stdlib.h>` และ `<time.h>`
- **ชี้แจงเบาๆ:** การประกาศ `char names[5][20]` คืออะไร
- **คำเตือนตัวยักษ์:** อย่าเอา `srand(time(NULL));` ไปใส่ไว้ใน Loop เด็ดขาด! (เวลาในคอมจะเปลี่ยนไม่ทัน ทำให้ได้เลขสุ่มซ้ำๆ รัวๆ)

**💡 เทคนิคการสอน:**
- ให้เด็กๆ เปลี่ยนชื่อใน Array เป็นชื่อเพื่อนตัวเอง 5 คน แล้วรันหาคนที่จะโดนเลี้ยงน้ำ!
- ให้น้องลองลบบรรทัด `srand(time(NULL));` ทิ้ง ปิดรันหลายๆ รอบ แล้วถามว่าเกิดอะไรขึ้น? (มันจะได้ผู้ชนะคนเดิมตลอดกาล)
- **โจทย์ท้าทาย (Super Boss):** "ใครเจ๋งสุดในห้อง ลองสุ่มจับรางวัลหาผู้โชคดี 3 คน โดยที่ **ห้ามซ้ำคนเดิม** ดูสิครับ!" (ข้อนี้ยากมาก ใบ้: ต้องสร้าง Array มาเก็บประวัติคนที่เคยโดนสุ่มไปแล้ว แล้วเช็คใน Loop)
